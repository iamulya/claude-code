## Purpose

The Doctor Tools subsystem provides a suite of code intelligence capabilities specifically for the YAAF Doctor agent. Its purpose is to equip an [Agent](../apis/agent.md) with the ability to interact with a YAAF-based project's source code by reading, searching, compiling, testing, and inspecting files [Source 1].

A key architectural feature is its security sandbox. All file and process operations executed by these tools are strictly confined to the developer's project root directory, preventing the agent from accessing or modifying files outside the intended project scope. These tools are designed to be general-purpose for any project that uses the YAAF framework, not just the YAAF source repository itself [Source 1].

## Architecture

The subsystem is centered around the `createDoctorTools` factory function. This function utilizes the core `buildTool` utility from the main [Tools](./tools.md) subsystem to construct a collection of `Tool` objects [Source 1].

Internally, the tools leverage standard Node.js modules to perform their actions:
- `fs/promises`: For asynchronous file system operations like reading files.
- `path`: For safe and cross-platform path manipulation within the project root.
- `child_process.execSync`: For executing shell commands, such as running a project's compilation or test scripts [Source 1].

The sandboxing is achieved by prepending the `projectRoot` path, provided during initialization, to all file paths accessed by the tools [Source 1].

## Integration Points

The primary consumer of this subsystem is the Doctor agent, a specialized agent within the [Doctor System](./doctor-system.md). The array of `Tool` objects generated by `createDoctorTools` is intended to be passed to the Doctor [Agent](../apis/agent.md) instance upon its creation, making these capabilities available to the LLM during its reasoning process.

## Key APIs

The main public API for this subsystem is:
- [createDoctorTools](../apis/create-doctor-tools.md): A factory function that takes a `projectRoot` string and returns an array of `Tool` objects ready for use by an agent [Source 1].

## Configuration

Configuration of the Doctor Tools is performed at runtime by passing the absolute path to the target project's root directory as an argument to the [createDoctorTools](../apis/create-doctor-tools.md) function. This `projectRoot` parameter defines the boundary for all subsequent tool operations [Source 1].
