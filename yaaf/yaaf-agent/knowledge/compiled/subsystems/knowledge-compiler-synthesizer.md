## Purpose

The [Knowledge Compiler](./knowledge-compiler.md) Synthesizer is responsible for the final assembly of a compiled knowledge base article. Its primary function is to process the raw markdown output from an [LLM](../concepts/llm.md), validate its [Frontmatter](../concepts/frontmatter.md) against the knowledge base [Ontology](../concepts/ontology.md), merge it with compiler-generated metadata, and serialize the final, complete frontmatter into a YAML block [Source 1].

This subsystem acts as a quality gate, ensuring that all generated articles have structurally correct and consistent metadata before being written to the knowledge base. The validation is designed to be strict for required fields and correct data types (e.g., enums) while being lenient for optional fields to avoid compilation failures over minor inconsistencies [Source 1].

## Architecture

The Synthesizer's [workflow](../concepts/workflow.md) is a sequential process orchestrated by several key functions within the `frontmatter.ts` module [Source 1]:

1.  **Parsing**: The process begins with `parseArticleOutput`, which takes the raw string generated by the LLM. It splits the string into a frontmatter object and the article body. If the LLM fails to include a frontmatter block, this function gracefully handles it by creating an empty frontmatter object [Source 1].

2.  **Validation**: The extracted frontmatter is then passed to `validateFrontmatter`. This function compares the LLM-provided values against the `FrontmatterSchema` defined in the [KBOntology](../apis/kb-ontology.md) for the given [Entity Type](../concepts/entity-type.md). It performs type coercion (e.g., from string to number or boolean) and validates that enum values and entity references (`entity_ref`) are valid according to the ontology and the ConceptRegistry [Source 1].

3.  **Merging**: The `buildCompleteFrontmatter` function merges data from three sources in a specific order of precedence:
    1.  LLM-generated and validated values.
    2.  Compiler-suggested frontmatter from the `ExtractorPlan`.
    3.  [Compiler-injected Metadata](../concepts/compiler-injected-metadata.md).

    Compiler-injected metadata always overrides other sources for its specific keys to ensure structural integrity. This metadata includes fields such as `entity_type`, `compiled_at`, `compiled_from`, `confidence`, and a `stub` flag [Source 1].

4.  **Serialization**: Finally, the complete, merged frontmatter object is passed to `serializeFrontmatter`, which converts it into a standard, `---`-delimited YAML string suitable for inclusion at the top of the final markdown file [Source 1].

## Integration Points

The Synthesizer is a late-stage component in the knowledge compilation pipeline and interacts with several other parts of the system:

*   **Knowledge Compiler**: The main compiler process invokes the Synthesizer to process LLM output for each article. It provides the `ExtractorPlan` and other compiler metadata required for the merging step.
*   **KBOntology**: The Synthesizer depends on the ontology subsystem to retrieve the `FrontmatterSchema` for a given entity type, which is the basis for all validation logic [Source 1].
*   **ConceptRegistry**: For validating `entity_ref` fields, the Synthesizer queries the ConceptRegistry to ensure that referenced document IDs exist in the knowledge base [Source 1].

## Key APIs

The public API of this subsystem is centered around the functions that execute the architectural workflow [Source 1].

*   `parseArticleOutput(raw: string): ParsedArticle`: Parses a raw markdown string from an LLM into a `ParsedArticle` object containing the frontmatter as a record and the body as a string.
*   `validateFrontmatter(rawValues, schema, entityType, registry, ontology)`: Validates a frontmatter object against the ontology schema. It returns a `FrontmatterValidationResult` indicating success or detailing any errors.
*   `buildCompleteFrontmatter(validatedValues, suggestedFrontmatter, compilerMeta)`: Merges validated LLM values, compiler suggestions, and injected metadata into a final, canonical frontmatter object.
*   `serializeFrontmatter(fields: Record<string, unknown>): string`: Serializes a given frontmatter object into a YAML string, including the `---` delimiters.
