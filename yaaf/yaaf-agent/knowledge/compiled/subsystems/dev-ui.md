## Purpose
The Dev UI Subsystem provides a local, web-based interface for developers to test, interact with, and debug YAAF agents. It is designed to be a "zero-dependency" tool, meaning all necessary assets (HTML, CSS, and JavaScript) are bundled into a single self-contained response. This eliminates the need for external frontend dependencies or complex build steps when a developer needs to inspect an agent's behavior during the development lifecycle.

## Architecture
The subsystem is architected to generate a complete HTML document at server startup, which is then cached and served. To maintain code quality and avoid the complexities of embedding large blocks of JavaScript and CSS within TypeScript template literals, the subsystem stores client-side logic and styles in separate files (`devUi.client.js` and `devUi.styles.css`). These files are read from the filesystem once at module load time and inlined into the final HTML string.

### Internal Components
*   **Tokenizer**: A lightweight (< 4KB) inline single-pass sticky-regex tokenizer that provides syntax highlighting for multiple languages, including JS/TS, Python, JSON, Bash/Shell, and CSS.
*   **Inspector**: A mobile-responsive interface component (implemented as a slide-up drawer on narrow viewports) that allows developers to inspect agent metadata, such as the model identifier and system prompts.
*   **Streaming Engine**: Supports partial Markdown previews during streaming responses, allowing developers to see agent output in real-time.

## Integration Points
The Dev UI is primarily integrated through the YAAF runtime's server implementation. When the `devUi: true` configuration is set, the `createServer()` function routes `GET /` requests to the string generated by this subsystem.

## Key APIs
The subsystem exposes a functional API for generating the interface:

### buildDevUiHtml
`buildDevUiHtml(opts: DevUiOptions): string`
This function assembles the final HTML string. It is typically called once during server initialization.

### DevUiOptions
A configuration object that defines the metadata and capabilities displayed in the UI:
*   `name`: The display name of the agent.
*   `version`: The current version of the agent.
*   `streaming`: A boolean indicating if the agent supports `runStream()`.
*   `model`: The model identifier shown in the inspector.
*   `multiTurn`: Indicates if server-side multi-turn history formatting is active.
*   `systemPrompt`: An optional string to display the agent's system prompt in a read-only settings view.

## Configuration
The Dev UI is configured via the `DevUiOptions` interface. It supports several "Phase 3" features that enhance the developer experience:
*   **Prompt Chips**: Welcome and empty states include prompt chips for quick interaction.
*   **Navigation**: Includes a floating "jump-to-bottom" button for long conversations.
*   **Export**: Provides functionality to export the current conversation as a Markdown (`.md`) file.
