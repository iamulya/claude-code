## What It Is
The Agent Project Structure is the standardized directory and file layout for a YAAF-based agent application [Source 1]. It is generated by the `yaaf init` command and provides a consistent, organized foundation for development. This structure separates the core concerns of an agent, such as its primary configuration, tool logic, instructional prompts ([Skills](./skills.md)), and personality, into distinct files and directories. This separation makes projects easier to navigate, maintain, and scale as complexity grows [Source 1].

## How It Works in YAAF
[when](../apis/when.md) a new project is created using the `yaaf init` command, YAAF scaffolds a predefined folder structure. This layout ensures that the framework's runtime and tooling can automatically discover and load components like [Tools](../subsystems/tools.md) and skills [Source 1].

The standard project structure is as follows [Source 1]:

```
my-agent/
├── src/
│   ├── agent.ts          # Agent configuration and entry point
│   └── tools/
│       └── search.ts     # Tool definitions
├── skills/
│   └── SKILL.md          # Agent skill instructions
├── SOUL.md               # Agent personality (optional)
├── tests/
│   └── agent.test.ts     # Test scaffold
├── package.json
├── tsconfig.json
├── .gitignore
└── README.md
```

### Key Files and Directories

The purpose of each key component in the structure is defined by convention [Source 1]:

| File / Directory | Purpose |
|------------------|---------|
| `src/agent.ts` | The main agent definition file. This is where the [System Prompt](./system-prompt.md), tool registrations, and model configuration are specified. |
| `src/tools/*.ts` | A directory containing tool definitions. Each file typically defines a single tool and exports the result of a `buildTool()` call. |
| `skills/*.md` | A directory for Markdown files that contain instructions, rules, or knowledge. These are injected into the agent's system prompt at runtime. |
| `SOUL.md` | An optional file that defines the agent's core personality, including its name, tone, and behavioral rules. This is used by certain components like `yaaf/gateway`. |
| `tests/` | Contains test scaffolds for the agent, such as `agent.test.ts`. |
| `.yaaf/[[Memory]]/` | A directory created at runtime by YAAF to manage [Memory Storage](./memory-storage.md) automatically. This directory should typically be added to `.gitignore`. |

## Configuration
The project structure is not configured via a file but is created and modified using the YAAF command-line interface ([CLI](../subsystems/cli.md)) [Source 1].

The initial structure is generated with the `init` command:
```bash
npx yaaf init my-agent
```

Developers can use other CLI commands to add new components, which will be placed in the correct directories according to the standard structure. For example, the `add` command scaffolds new files in the `src/tools/` or `skills/` directories [Source 1]:

```bash
# Adds a new tool file to src/tools/
yaaf add tool weather

# Adds a new skill file to skills/
yaaf add skill code-review
```
