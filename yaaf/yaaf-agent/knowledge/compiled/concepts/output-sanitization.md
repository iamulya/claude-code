## What It Is
Output Sanitization is a security layer in YAAF designed to scrub and validate content generated by Large Language Models (LLMs) before it is processed by downstream systems or rendered in user interfaces. Because LLMs are non-deterministic and can be manipulated via prompt injection or simply generate malformed content, raw LLM output is treated as untrusted data.

This concept addresses several specific security and stability risks:
*   **Cross-Site Scripting (XSS):** Preventing the execution of malicious scripts in web-based dashboards or chat interfaces.
*   **Resource Exhaustion:** Preventing "denial of service" scenarios where an LLM generates an unexpectedly massive response that consumes excessive memory or storage.
*   **Malicious Redirects:** Identifying and neutralizing dangerous URI schemes (e.g., `javascript:`, `data:`) that could be used for phishing or local file access.

## How It Works in YAAF
In YAAF, output sanitization is primarily handled by the `OutputSanitizer` class found in the `security` module. It operates by applying a series of configurable rules to the text returned by an LLM.

### Core Mechanisms
*   **HTML/XSS Stripping:** The sanitizer can remove specific dangerous tags (like `<script>` or `<iframe>`) and event handlers (like `onclick`), or it can be configured to strip all HTML tags entirely.
*   **Markdown Sanitization:** It can identify and remove raw HTML blocks embedded within Markdown (e.g., ` ```html ` blocks) to ensure the output remains pure text or safe Markdown.
*   **URL Validation:** The system flags or removes suspicious URLs, specifically targeting non-standard protocols like `javascript:`, `data:`, and `vbscript:`.
*   **Content Length Limits:** To prevent memory exhaustion, the sanitizer enforces a `maxLength` (defaulting to 100,000 characters), truncating any content that exceeds the budget.

### Integration Patterns
Developers can implement sanitization in three primary ways:
1.  **Hooks:** Using the `afterLLM` hook to automatically intercept and clean every response generated by an agent.
2.  **Standalone Utility:** Calling the `.sanitize()` method directly on any string.
3.  **Safe Wrappers:** Using `safeRun()` as a wrapper around `agent.run()` to ensure the final output is processed.

When a modification occurs, the sanitizer generates a `SanitizeEvent`, which includes the type of modification (e.g., `html_stripped`, `truncated`) and the number of changes made. This allows for auditing and logging of potentially malicious LLM behavior.

## Configuration
The `OutputSanitizer` is highly configurable via the `OutputSanitizerConfig` object. Developers can choose between standard sanitization or a "strict" mode that removes all HTML.

```ts
import { OutputSanitizer, Agent } from 'yaaf';

// Initialize with custom configuration
const sanitizer = new OutputSanitizer({
  stripHtml: true,
  sanitizeUrls: true,
  maxLength: 50000,
  stripMarkdownHtml: true,
  onSanitize: (event) => {
    console.log(`Sanitization event: ${event.type} (${event.count} changes)`);
  }
});

// Apply as a hook to an Agent
const agent = new Agent({
  hooks: {
    afterLLM: sanitizer.hook(),
  },
});

// Manual usage
const result = sanitizer.sanitize('<script>alert("xss")</script>Safe Content');
// result.text -> "Safe Content"
// result.modified -> true
```

### Configuration Options
| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `stripHtml` | boolean | `true` | Removes all HTML tags. |
| `stripDangerousHtml` | boolean | `true` | Removes only scripts, iframes, and event handlers (ignored if `stripHtml` is true). |
| `sanitizeUrls` | boolean | `true` | Removes `javascript:`, `data:`, and `vbscript:` URLs. |
| `maxLength` | number | `100_000` | Truncates output beyond this character count. |
| `stripMarkdownHtml` | boolean | `false` | Strips raw HTML blocks from within Markdown. |
| `customSanitizer` | function | `undefined` | A custom function for additional domain-specific cleaning. |
