## What It Is

Audit Logging in YAAF is the process of systematically recording security-relevant events generated by the framework's security subsystems. Specifically, it captures the outcomes of integrity and trust verifications performed by the TrustPolicy system [Source 1]. These logs create an audit trail that can be used for monitoring agent behavior, debugging security issues, ensuring compliance, and maintaining accountability for actions taken by or through the agent, such as loading a plugin or using a tool from an MCP server [Source 1].

## How It Works in YAAF

The primary mechanism for audit logging is a callback function provided during the configuration of a `TrustPolicy` instance. The framework invokes this callback for every verification event that occurs [Source 1].

The callback receives a `TrustVerificationEvent` object, which contains detailed information about the security decision. The structure of this event is as follows [Source 1]:

*   `target`: The type of entity being verified (e.g., `"plugin"`, `"mcp_server"`, `"mcp_tool"`).
*   `name`: The identifier of the entity (e.g., the plugin name).
*   `result`: The outcome of the verification (e.g., `"trusted"`, `"verified"`, `"warning"`, `"blocked"`, `"unknown"`).
*   `reason`: A human-readable string explaining why the result was reached.
*   `timestamp`: A `Date` object indicating [when](../apis/when.md) the event occurred.

By implementing this callback, developers can pipe these structured security events into any logging, monitoring, or security information and event management (SIEM) system of their choice. The framework provides the event data, and the developer is responsible for its storage and processing [Source 1].

## Configuration

Audit logging is configured via the `onVerification` property within the `TrustPolicyConfig` object when creating a `TrustPolicy`. A developer provides a function that accepts a `TrustVerificationEvent` and processes it as needed [Source 1].

The following example demonstrates how to configure a simple audit logger that prints verification events to the console:

```typescript
import { TrustPolicy, TrustPolicyConfig, TrustVerificationEvent } from 'yaaf';

// Define a function to handle audit events
const consoleAuditLogger = (event: TrustVerificationEvent) => {
  console.log(
    `[SECURITY_AUDIT] ${event.timestamp.toISOString()} | ` +
    `Target: ${event.target} | Name: ${event.name} | ` +
    `Result: ${event.result} | Reason: ${event.reason}`
  );
};

// Configure the TrustPolicy to use the logger
const policyConfig: TrustPolicyConfig = {
  mode: 'strict',
  plugins: {
    'my-plugin': { sha256: 'abc123...' },
  },
  onVerification: consoleAuditLogger,
};

const trustPolicy = new TrustPolicy(policyConfig);

// Any verification performed by this trustPolicy instance will now
// trigger the consoleAuditLogger function.
```
