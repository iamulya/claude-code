## What It Is

Streaming is a core capability in YAAF that allows an agent to provide real-time feedback and incremental output during its execution. Instead of waiting for the entire thought process, [Tool Execution](./tool-execution.md), and final response generation to complete, a streaming-enabled agent emits a sequence of events as they occur. This provides a more interactive and transparent user experience, especially in conversational interfaces like command-line [Tools](../subsystems/tools.md), by showing token-by-token text generation and the status of [Tool Calls](./tool-calls.md) in real time [Source 1].

## How It Works in YAAF

The primary mechanism for enabling streaming is the `toStreamableAgent()` adapter function. This function takes a standard YAAF `Agent` instance and returns a `StreamableAgent` wrapper. While the standard `run()` method remains available for a complete, single response, the wrapped agent also exposes a `runStream()` method [Source 1].

The `runStream()` method returns an `AsyncIterable<RuntimeStreamEvent>`, which a runtime or client can iterate over to receive live updates from the agent's execution process. YAAF defines several types of `RuntimeStreamEvent` to represent different stages of execution [Source 1]:

| Event Type | Description |
|---|---|
| `text_delta` | A small chunk of text (often a single token) generated by the [LLM](./llm.md). |
| `tool_call_start` | Signals that a tool is about to be executed, including its name and arguments. |
| `tool_call_end` | Signals that a tool has finished executing, including its duration and any errors. |
| `tool_blocked` | Indicates that a tool call was attempted but denied due to permissions or other constraints. |
| `usage` | Provides token usage statistics for the current run. |
| `done` | The final event in the stream, containing the complete, aggregated text response. |

This event stream is consumed by YAAF's built-in runtimes, such as `create[[[[[[[[CLI]]]]]]]]` and `createInkCLI`, to render live progress indicators and append text to the user's console as it is generated [Source 1].

## Configuration

To use streaming, a standard `Agent` must first be passed to the `toStreamableAgent` adapter. Runtimes that support streaming, like `createCLI`, can then be configured to use this capability.

The following example demonstrates adapting an agent and enabling streaming in the `createCLI` runtime [Source 1]:

```typescript
import { Agent, toStreamableAgent } from 'yaaf';
import { createCLI } from 'yaaf/cli-runtime';

// 1. Create a standard agent
const agent = new Agent({
  systemPrompt: 'You are a helpful assistant.',
  // ... other agent configuration
});

// 2. Adapt the agent to make it streamable
const streamableAgent = toStreamableAgent(agent);

// 3. Pass the streamable agent to a runtime and enable streaming
createCLI(streamableAgent, {
  name: 'my-assistant',
  greeting: '👋 Hello! How can I help?',
  
  // Enable streaming mode
  streaming: true,
});
```

In this configuration, [when](../apis/when.md) the user interacts with the CLI, the `createCLI` runtime will call `streamableAgent.runStream()` and render the `RuntimeStreamEvent` objects as they are received [Source 1].
