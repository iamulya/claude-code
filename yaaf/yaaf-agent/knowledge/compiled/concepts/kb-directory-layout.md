## What It Is
The KB Directory Layout is a convention-based filesystem structure that the YAAF Knowledge Base (KB) pipeline uses to manage the lifecycle of information. It provides a predictable organization for raw source materials, the resulting LLM-synthesized wiki articles, and the metadata required for incremental compilation and quality assurance. By enforcing a specific layout, YAAF ensures that the `KBCompiler` can reliably wire together the ingestion, extraction, synthesis, and linting stages.

## How It Works in YAAF
The `KBCompiler` class acts as the coordinator for the KB pipeline and manages the directory layout. All paths within the layout are relative to a root directory defined by the developer (the `kbDir`).

### Core Components
The layout is divided into input sources, generated outputs, and system metadata:

*   **`ontology.yaml`**: A required file loaded at initialization. It defines the domain schema that guides the extraction and synthesis processes.
*   **`raw/`**: The directory for source files, often referred to as "raw materials." This folder contains the primary inputs for the `Ingester`. Common subdirectories include:
    *   `papers/`: Research papers in formats such as `.md` or `.html`.
    *   `web-clips/`: Content captured from the web (e.g., Obsidian Web Clipper output).
    *   `tools/`: Documentation regarding specific tools.
    *   `datasets/`: Descriptions and metadata for datasets.
*   **`compiled/`**: The output directory for the LLM-authored wiki. This area is managed by the `Synthesizer` and is organized into subdirectories based on entity types:
    *   `concepts/`: Articles regarding core abstractions and principles.
    *   `tools/`: Articles regarding specific tool implementations.
    *   `research-papers/`: Synthesized summaries of research papers.
*   **`.kb-registry.json`**: An automatically maintained index cache. It tracks the relationship between source files and compiled articles to support incremental compilation.
*   **`.kb-lint-report.json`**: The report generated by the `KBLinter` during the last run, used to track quality issues and facilitate CI/CD checks.

## Configuration
Developers configure the directory layout by providing the `kbDir` path during the creation of the `KBCompiler`.

```ts
const compiler = await KBCompiler.create({
  kbDir: '/path/to/my-kb',
  extractionModel: makeGenerateFn(new GeminiChatModel({ model: 'gemini-2.5-flash' })),
  synthesisModel:  makeGenerateFn(new GeminiChatModel({ model: 'gemini-2.5-pro' })),
})

const result = await compiler.compile()
```

The compiler uses this directory to locate the `ontology.yaml` and the `raw/` folder, and it will automatically create or update the `compiled/` directory and metadata files during the execution of the `compile()` method.
