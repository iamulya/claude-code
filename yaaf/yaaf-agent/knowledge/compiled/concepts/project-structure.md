## What It Is

The YAAF Agent Project Structure is the standardized directory and file layout for an agent created with the YAAF framework [Source 1]. This convention provides a consistent and predictable organization for an agent's source code, configuration, and related assets. It is designed to simplify project navigation, facilitate maintenance, and enable YAAF's tooling, such as the [YAAF CLI](./yaaf-cli.md), to automatically discover and manage components like tools and skills [Source 1].

## How It Works in YAAF

The standard project structure is generated by the `yaaf init` command [Source 1]. This command scaffolds a new project with a predefined set of directories and files, establishing a clear separation of concerns for different parts of the agent.

A typical YAAF project has the following layout [Source 1]:

```
my-agent/
├── src/
│   ├── agent.ts          # Agent configuration and entry point
│   └── tools/
│       └── search.ts     # Tool definitions
├── skills/
│   └── SKILL.md          # Agent skill instructions
├── SOUL.md               # Agent personality (optional)
├── tests/
│   └── agent.test.ts     # Test scaffold
├── package.json
├── tsconfig.json
├── .gitignore
└── README.md
```

### Key Files and Directories

The purpose of each key file and directory is as follows [Source 1]:

| Path                | Purpose                                                                                             |
| ------------------- | --------------------------------------------------------------------------------------------------- |
| `src/agent.ts`      | The main entry point for the agent. This file contains the primary agent configuration, including the [System Prompt](./system-prompt.md), tool registrations, and model settings. |
| `src/tools/*.ts`    | A directory containing tool definitions. The convention is to have one file per tool, with each file exporting the result of a `buildTool()` call. |
| `skills/*.md`       | A directory for markdown files that define [Agent Skills](./agent-skills.md). The content of these files is automatically injected into the agent's system prompt. |
| `SOUL.md`           | An optional file that defines the agent's personality, including its name, tone, and behavioral rules, as part of the [Agent Personality System (SOUL)](./agent-personality-system-soul.md). |
| `.yaaf/memory/`     | A directory created at runtime for auto-managed [Memory Storage](./memory-storage.md). This directory is typically excluded from version control. |
| `tests/`            | Contains test files for the agent, with a scaffold typically provided for `agent.ts`. |
