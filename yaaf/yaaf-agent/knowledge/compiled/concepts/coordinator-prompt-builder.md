## What It Is
The CoordinatorPromptBuilder is a component within YAAF's [Coordinator Mode](../subsystems/coordinator-mode.md) subsystem responsible for dynamically generating the [System Prompt](./system-prompt.md) for a [CoordinatorAgent](./coordinator-agent.md) [Source 1]. This generated prompt is a critical piece of [Prompt Engineering](./prompt-engineering.md) that instructs the [CoordinatorAgent](./coordinator-agent.md) on how to manage a team of [worker agents](./worker-agent.md).

The prompt provides the [CoordinatorAgent](./coordinator-agent.md) with all the necessary context to delegate tasks effectively and synthesize results. Specifically, it includes [Source 1]:
*   **Worker Capabilities**: A list of available [worker agents](./worker-agent.md), their unique IDs, descriptions of their specializations, and the tools they can use.
*   **Concurrency Rules**: Guidelines on how to manage parallel task execution.
*   **Synthesis Guidance**: Instructions on how to combine the outputs from multiple workers into a coherent, final response for the user.

## How It Works in YAAF
The functionality of the CoordinatorPromptBuilder is primarily implemented by the [buildCoordinatorPrompt](../apis/build-coordinator-prompt.md) API function [Source 1]. This function takes a configuration object (`CoordinatorPromptConfig`) containing the details of the worker agents and assembles a complete [System Prompt](./system-prompt.md) string.

This process involves formatting the worker information and combining it with a pre-defined instruction template that guides the [CoordinatorAgent](./coordinator-agent.md)'s reasoning process. The resulting string is then used as the system message in all subsequent [LLM calls](./llm-call.md) made by that [CoordinatorAgent](./coordinator-agent.md) instance.

The prompt engineering logic in YAAF is a distilled and simplified adaptation of a more extensive, battle-tested coordinator prompt, focusing on the essential patterns that make the coordinator-worker pattern effective [Source 1].

## Configuration
The behavior of the CoordinatorPromptBuilder is configured by providing a list of workers when creating a [CoordinatorAgent](./coordinator-agent.md). Each worker's `id`, `description`, and `tools` are used by the builder to construct the capabilities section of the prompt.

The following example shows how workers are defined, which in turn serves as the configuration for the prompt builder [Source 1]:

```typescript
const coordinator = createCoordinator({
  model,
  workers: [
    { id: 'researcher', tools: [searchTool, readTool], description: 'Research agent for finding information.' },
    { id: 'implementer', tools: [editTool, writeTool], description: 'Implementation agent for writing and modifying code.' },
  ],
});
```

In this case, the CoordinatorPromptBuilder would generate a [System Prompt](./system-prompt.md) informing the [CoordinatorAgent](./coordinator-agent.md) that it has access to a `researcher` and an `implementer`, and it would detail the specific tools available to each.
