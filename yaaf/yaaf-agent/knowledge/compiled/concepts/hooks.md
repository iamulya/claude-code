## What It Is
Hooks are lifecycle callbacks integrated into the YAAF agent execution loop. They serve as interceptors for critical operations, specifically tool invocations and Large Language Model (LLM) turns. 

Unlike standard events, which are read-only and used for observation, hooks are active participants in the agent's control flow. They provide developers with the ability to block execution, modify inputs/outputs, or short-circuit the agent's logic. This makes them essential for implementing safety guards, human-in-the-loop patterns, and detailed audit logging.

## How It Works in YAAF
Hooks are executed at specific stages of the agent's run loop. When a hook is triggered, the framework passes a `HookContext` object to the callback, providing stateful information about the current execution.

### Hook Context
The `HookContext` provides the following metadata to the callback:
*   **toolName**: The name of the tool currently being targeted.
*   **arguments**: The parsed arguments generated by the LLM for the tool call.
*   **messages**: The full conversation history (as an array of `ChatMessage`) at the time the hook is invoked.
*   **iteration**: The current count of LLM calls within the specific agent run, useful for preventing infinite loops or managing token budgets.

### Execution Flow
The framework supports different types of hooks depending on the stage of execution:
*   **beforeToolCall**: Executed after the LLM has requested a tool call but before the tool's underlying function is invoked. This hook can return a `block` action to prevent the tool from running (e.g., for permission checks).
*   **afterToolCall**: Executed after the tool has finished its operation. This allows for the inspection or modification of the tool's result before it is fed back into the conversation history.

Hooks must return a result object, such as `HookResult` or `LLMHookResult`. If the hook returns `{ action: 'continue' }`, the agent proceeds with its normal execution. If it returns a `block` action (as demonstrated in the framework's implementation examples), the execution is halted.

## Configuration
Hooks are defined during the instantiation of an `Agent` via the configuration object.

```ts
const agent = new Agent({
  systemPrompt: '...',
  hooks: {
    beforeToolCall: async ({ toolName, arguments: args }) => {
      // Example: Implement a manual approval step for sensitive tools
      if (toolName === 'book_trip' && !userConfirmed) {
        return { action: 'block', reason: 'Awaiting user confirmation' };
      }
      return { action: 'continue' };
    },
    afterToolCall: async ({ toolName }, result) => {
      // Example: Record tool results to an external audit log
      await auditLog.record({ tool: toolName, result });
      return { action: 'continue' };
    },
  },
});
```
