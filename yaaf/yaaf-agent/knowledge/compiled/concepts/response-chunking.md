## What It Is

Response Chunking is the process of splitting a single, long text response generated by an agent into multiple, smaller, sequential messages. This is necessary because many communication platforms, such as Discord or Telegram, impose a maximum character limit on individual messages. If an agent's response exceeds this limit, it would either be truncated or fail to send entirely.

Response Chunking solves this problem by dividing the content into appropriately sized parts that respect the target [Channel](../apis/channel.md)'s constraints, ensuring the complete response is delivered reliably to the user as a series of messages [Source 1].

## How It Works in YAAF

In YAAF, response chunking is primarily handled by the `Gateway` subsystem, which routes messages between the agent and various communication channels [Source 1]. The process occurs after the agent has generated its complete response but before it is sent to the user's channel.

The core mechanism is the `afterProcess` hook within the `GatewayConfig`. This function receives the full agent response string and can return an array of strings (`string[]`). [when](../apis/when.md) an array is returned, the `Gateway` interprets each string as a separate message to be sent sequentially through the channel [Source 1].

YAAF provides a utility function, `chunkResponse(text: string, channelName: string)`, designed for this purpose. This function intelligently splits text at paragraph or sentence boundaries to avoid breaking words or sentences in the middle. It uses channel-specific `ChannelLimits`, such as `maxLength`, to determine the appropriate size for each chunk [Source 1]. While developers can implement their own logic, this function provides a standard, robust solution.

## Configuration

Developers can customize or override the default chunking behavior by providing a custom `afterProcess` function in the `Gateway` configuration. This allows for fine-grained control over how responses are split.

For example, a developer could implement a simple chunking strategy or use the provided `chunkResponse` utility.

```typescript
import { Gateway, chunkResponse, InboundMessage } from 'yaaf';

// Assume 'agent' and 'channels' are already defined.

const gateway = new Gateway({
  agent,
  channels,
  
  /**
   * Custom hook to handle response processing.
   * This implementation uses the built-in chunkResponse utility.
   */
  afterProcess: (response: string, message: InboundMessage): string[] => {
    // The channel name is available from the original inbound message context.
    const channelName = message.channel; 
    
    // Use the utility to split the response into chunks suitable for the channel.
    return chunkResponse(response, channelName);
  },
});

// When the gateway processes a message, this hook will automatically
// split any long responses before sending them.
```
In this example, the `afterProcess` function takes the full `response` from the agent and the original `InboundMessage`. It then calls `chunkResponse` to split the text according to the limits of the originating channel, returning an array of strings for the `Gateway` to send [Source 1].
