## What It Is

A JSON Compilation Plan is a structured JSON object that serves as the primary output of the [Concept Extractor](../subsystems/concept-extractor.md) subsystem [Source 1]. It functions as an actionable blueprint for the knowledge base compiler, specifying which articles to create or update based on ingested source material. The plan also defines the relationships between these articles, forming the basis for [Wikilinks](./wikilinks.md) and the overall knowledge graph structure [Source 1].

This concept solves the problem of translating unstructured or semi-structured source text into a formal, machine-readable set of instructions for populating and maintaining the YAAF knowledge base.

## How It Works in YAAF

The JSON Compilation Plan is generated by an LLM (typically a fast and cost-effective model like `gemini-2.5-flash`) during the [Concept Extractor](../subsystems/concept-extractor.md)'s classification pass [Source 1]. This process is initiated by a detailed prompt constructed by the [buildExtractionUserPrompt](../apis/build-extraction-user-prompt.md) function.

The LLM synthesizes the following inputs to produce the plan [Source 1]:
1.  **Ontology**: The definitions for each [Entity Type](./entity-type.md) and their required [Frontmatter](./frontmatter.md) schemas, which ground the LLM in the target data model.
2.  **Concept Registry**: A list of all articles that already exist in the knowledge base, providing the "known universe" of concepts.
3.  **Vocabulary**: A set of canonical terms and their known aliases, helping the LLM to correctly identify and link entities.
4.  **Static Analysis**: Pre-computed facts about the source text, such as entity mentions and directory structure hints, which the LLM can use as reliable information.
5.  **Ingested Content**: The source text itself, which is the primary subject of the analysis.

The resulting JSON object contains a list of proposed actions—such as creating a new article with specific frontmatter or updating an existing one—that are then executed by subsequent stages of the knowledge ingestion pipeline.
