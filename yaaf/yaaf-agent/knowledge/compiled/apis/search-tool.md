## Overview

The `searchTool` is a sample tool provided as part of the project structure generated by the `yaaf init` command-line interface [Source 1]. It is not a functional web search tool but serves as a placeholder and a practical example for developers on how to define a custom tool using the [buildTool](./build-tool.md) factory function [Source 1].

Its primary purpose is to demonstrate the basic structure of a tool, including its name, description, input schema, and an `execute` function. Developers are expected to replace the placeholder implementation in the `execute` function with a connection to a real search API, such as Brave, Tavily, or SearXNG [Source 1].

## Examples

The definition of `searchTool` itself serves as the primary example of its usage. When a new YAAF project is created with `yaaf init`, a file containing this tool is generated. A developer would then modify the `execute` method to integrate a real search service.

### Default Scaffolded Tool

This is the code as it appears in a newly scaffolded project. It demonstrates the required structure for a tool but returns a static, placeholder string.

```typescript
// File: src/tools/search.ts (in a scaffolded project)

import { buildTool } from 'yaaf';

export const searchTool = buildTool({
  name: 'search',
  description: 'Search the web for information. Use this when the user asks about current events or facts you are unsure about.',
  inputSchema: {
    type: 'object',
    properties: {
      query: {
        type: 'string',
        description: 'The search query',
      },
    },
    required: ['query'],
  },
  execute: async (input) => {
    // This part should be replaced by the developer.
    console.log(`Searching for: ${input.query}`);
    return `Search results for: "${input.query}"\n\n[Replace this with a real search integration]`;
  },
});
```
