## Overview

The `deduplicatePlans` function is a step in the knowledge compilation process that detects and merges near-duplicate `ArticlePlan` objects [Source 1]. It operates after plans have been extracted from source material but before the synthesis stage where [LLM](../concepts/llm.md)s are invoked. Its primary purpose is to prevent redundant work and content fragmentation by consolidating topics that are semantically similar [Source 1].

This function helps avoid:
1.  Redundant [LLM Call](../concepts/llm-call.md)s for nearly identical topics.
2.  The creation of duplicate compiled articles with slightly different titles.
3.  The splitting of relevant source material across multiple similar articles.

`deduplicatePlans` uses three heuristics that do not require an LLM Call [Source 1]:
*   **Title Similarity**: [Jaccard Similarity](../concepts/jaccard-similarity.md) on the tokenized, lowercased titles of two plans.
*   **Source Path Overlap**: The ratio of shared source file paths between two plans.
*   **Description Similarity**: Jaccard similarity on the description text, which helps identify articles about the same concept even if their titles have low overlap (e.g., "BERT Architecture" and "Bidirectional Encoder Representations") [Source 1].

The underlying algorithm uses a Union-Find data structure to handle transitive duplicates. If plan A is similar to B, and B is similar to C, all three will be merged into a single group, even if A and C do not meet the similarity threshold directly. This is an improvement over a previous greedy algorithm that might miss such transitive relationships [Source 1].

Within a group of merged plans, a single "survivor" is chosen deterministically based on the lexicographically smallest `docId` [Source 1].

## Signature

The function takes an array of `ArticlePlan` objects and an optional similarity threshold. It returns a `DedupResult` object containing the surviving plans and a record of which plans were merged.

```typescript
import type { ArticlePlan } from "./extractor/index.js";

export interface DedupResult {
  /** Surviving plans after deduplication */
  merged: ArticlePlan[];
  /** Plans that were absorbed into survivors */
  removed: Array<{ docId: string; mergedInto: string; similarity: number }>;
}

export function deduplicatePlans(
  plans: ArticlePlan[],
  threshold?: number, // default: 0.7
): DedupResult;
```

### Parameters

*   `plans` (`ArticlePlan[]`): An array of article plans generated by the extractor stage.
*   `threshold` (`number`, optional): The minimum combined similarity score required to consider two plans duplicates. Defaults to `0.7` [Source 1].

### Return Value

*   (`DedupResult`): An object containing two properties:
    *   `merged`: An array of the surviving `ArticlePlan` objects after merging.
    *   `removed`: An array of objects detailing which plans were removed and which survivor they were merged into.

## Examples

The following example demonstrates how `deduplicatePlans` handles transitive relationships. Plan A ("BERT") is similar to Plan B ("Bidirectional..."), and Plan B is similar to Plan C ("Transformers"). The function correctly identifies them as a single conceptual group and merges them all into the plan with the lexicographically smallest `docId` (`doc-01`).

```typescript
import { deduplicatePlans } from 'yaaf';
import type { ArticlePlan } from 'yaaf';

// A simplified ArticlePlan for demonstration purposes
type SimpleArticlePlan = {
  docId: string;
  title: string;
  description: string;
  sourcePaths: string[];
};

const planA: SimpleArticlePlan = {
  docId: 'doc-01',
  title: 'BERT Architecture',
  description: 'A deep dive into the pre-trained language model from Google.',
  sourcePaths: ['/sources/bert.pdf'],
};

const planB: SimpleArticlePlan = {
  docId: 'doc-02',
  title: 'Bidirectional Encoder Representations from Transformers',
  description: 'An overview of the pre-trained language model known as BERT.',
  sourcePaths: ['/sources/bert.pdf', '/sources/transformers.md'],
};

const planC: SimpleArticlePlan = {
  docId: 'doc-03',
  title: 'Understanding Transformers',
  description: 'The core architecture behind models like BERT and GPT.',
  sourcePaths: ['/sources/transformers.md'],
};

const allPlans: SimpleArticlePlan[] = [planA, planB, planC];

const { merged, removed } = deduplicatePlans(allPlans, 0.7);

console.log('Surviving Plans:', merged.map(p => p.docId));
// Expected output:
// Surviving Plans: [ 'doc-01' ]

console.log('Removed Plans:', removed);
// Expected output:
// Removed Plans: [
//   { docId: 'doc-02', mergedInto: 'doc-01', similarity: 0.85 },
//   { docId: 'doc-03', mergedInto: 'doc-01', similarity: 0.78 }
// ]
// Note: Similarity values are illustrative.
```
