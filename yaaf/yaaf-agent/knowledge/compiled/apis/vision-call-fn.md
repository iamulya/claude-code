## Overview

`VisionCallFn` is a TypeScript type definition for an asynchronous function that interacts with a vision-capable Large Language Model ([LLM](../concepts/llm.md)). It defines the contract for functions that can process both text and image inputs simultaneously. [Source 1]

This type is used for multimodal tasks where an agent needs to analyze or interpret visual information, such as describing a screenshot, identifying objects in a picture, or answering questions about a diagram. The function signature requires the image to be provided as a base64-encoded string along with its corresponding MIME type. [Source 1]

Functions of this type are typically created using the `makeKBVisionClient` factory function. [Source 1]

## Signature

`VisionCallFn` is a function type that accepts a single object parameter and returns a `Promise<string>`.

```typescript
export type VisionCallFn = (params: {
  system: string;
  user: string;
  imageBase64: string;
  imageMimeType: string;
  temperature?: number;
  maxTokens?: number;
}) => Promise<string>;
```

### Parameters

The function receives a single object with the following properties:

- `system` [string]: The [System Prompt](../concepts/system-prompt.md), providing high-level instructions or context to the LLM.
- `user` [string]: The user's text prompt or question related to the image.
- `imageBase64` [string]: The image data, encoded as a base64 string.
- `imageMimeType` [string]: The MIME type of the image (e.g., `image/png`, `image/jpeg`).
- `temperature` [number] (optional): Controls the randomness of the model's output.
- `maxTokens` [number] (optional): The maximum number of tokens to generate in the response.

### Return Value

- `Promise<string>`: A promise that resolves to the string response generated by the LLM.

## Examples

The following example demonstrates how to create a vision client using `makeKBVisionClient` and then use the resulting function, which conforms to the `VisionCallFn` type, to analyze a local image file.

```typescript
import { makeKBVisionClient, VisionCallFn } from 'yaaf';
import * as fs from 'fs/promises';

// This function demonstrates the usage of a VisionCallFn
async function analyzeScreenshot(filePath: string) {
  // makeKBVisionClient returns a function of type VisionCallFn
  const visionClient: VisionCallFn = makeKBVisionClient({
    provider: 'openai', // Or 'gemini', 'anthropic'
    // API key is typically read from environment variables
  });

  try {
    const imageBuffer = await fs.readFile(filePath);
    const imageBase64 = imageBuffer.toString('base64');

    const response = await visionClient({
      system: 'You are a UI/UX expert. Analyze the provided screenshot.',
      user: 'What is the primary call-to-action on this screen and suggest one improvement.',
      imageBase64: imageBase64,
      imageMimeType: 'image/png', // Adjust based on the actual file type
      temperature: 0.2,
    });

    console.log('LLM Analysis:', response);
  } catch (error) {
    console.error('Failed to analyze screenshot:', error);
  }
}

// Usage:
// analyzeScreenshot('path/to/your/screenshot.png');
```
