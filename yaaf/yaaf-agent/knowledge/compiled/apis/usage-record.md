## Overview

The `UsageRecord` type defines the data structure for capturing the token counts from a single interaction with a Large Language Model ([LLM](../concepts/llm.md)). It is primarily used as the input for the `CostTracker.record()` method to log token consumption [Source 1].

This type includes mandatory fields for input (prompt) and output (completion) tokens, as well as optional fields for tracking tokens read from or written to a cache. This allows for detailed accounting of token usage, which is essential for cost estimation and performance monitoring [Source 1].

## Signature

`UsageRecord` is a TypeScript type alias with the following structure [Source 1]:

```typescript
export type UsageRecord = {
  inputTokens: number;
  outputTokens: number;
  cacheReadTokens?: number;
  cacheWriteTokens?: number;
};
```

### Fields

-   `inputTokens`: The number of tokens sent to the model in the prompt.
-   `outputTokens`: The number of tokens generated by the model in the completion.
-   `cacheReadTokens` (optional): The number of tokens that were served from a cache instead of being generated by the model.
-   `cacheWriteTokens` (optional): The number of tokens written to a cache during the interaction.

## Examples

A `UsageRecord` object is typically created to represent the outcome of a single LLM API call and then passed to a `CostTracker` instance.

```typescript
import { CostTracker, UsageRecord } from 'yaaf';

// Assume CostTracker is instantiated
const tracker = new CostTracker();

// Create a usage record for a call that did not use caching
const usage: UsageRecord = {
  inputTokens: 1000,
  outputTokens: 500,
};

// Record the usage for a specific model
tracker.record('gpt-4o', usage);

// Example with cache usage
const cachedUsage: UsageRecord = {
  inputTokens: 250,       // Tokens sent to the model
  outputTokens: 150,      // Tokens generated by the model
  cacheReadTokens: 4000,  // Tokens retrieved from a semantic cache
};

tracker.record('claude-3-opus', cachedUsage);
```
