## Overview

The `ParsedBody` type is a normalized data structure that represents the `requestBody` of an operation within an OpenAPI 3.x specification [Source 2]. It is an intermediate representation generated by YAAF's internal [OpenAPI Parser](../subsystems/open-api-parser.md). This type is not typically instantiated directly by users but is exposed as part of the `ParsedOperation` type for advanced use cases or introspection [Source 1, Source 2].

`ParsedBody` encapsulates the essential details about an API operation's payload, including its requirement status, media type (e.g., `application/json`), and the JSON Schema that defines its structure and constraints [Source 2]. This information is used by the `OpenAPI[[[[[[[[Tools]]]]]]]]et` to correctly construct and validate API calls made by an agent.

## Signature

`ParsedBody` is a TypeScript type alias with the following structure [Source 2]:

```typescript
export type ParsedBody = {
  /**
   * Whether the request body is required for the operation.
   */
  required: boolean;

  /**
   * The media type (MIME type) of the request body, e.g., 'application/json'.
   */
  mediaType: string;

  /**
   * The JSON Schema object describing the structure of the request body.
   */
  schema: Record<string, unknown>;
};
```

## Examples

### Conceptual Representation

The `ParsedBody` type is generated by the `parseOpenAPISpec` function from an OpenAPI document. Given the following `requestBody` definition in an OpenAPI specification:

```yaml
# Part of an OpenAPI operation definition
requestBody:
  description: Pet object that needs to be added to the store
  required: true
  content:
    application/json:
      schema:
        $ref: '#/components/schemas/Pet'
```

The YAAF parser would produce a `ParsedBody` object that looks like this (assuming the `$ref` to `Pet` has been resolved):

```typescript
const parsedBody: ParsedBody = {
  required: true,
  mediaType: 'application/json',
  schema: {
    type: 'object',
    properties: {
      name: { type: 'string', example: 'doggie' },
      photoUrls: {
        type: 'array',
        items: { type: 'string' }
      }
    },
    required: ['name', 'photoUrls']
  }
};
```

This `parsedBody` object would then be attached as the `requestBody` property of its corresponding `ParsedOperation` object.
