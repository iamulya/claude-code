## Overview

`RunnerStreamEvent` is a union type representing the detailed, internal stream events yielded by the [AgentRunner](./agent-runner.md) during an agent's execution cycle [Source 1]. These events provide a granular, real-time view of the agent's internal state changes, including text generation, tool interactions, LLM requests, and iteration cycles.

This type is intended for advanced use cases or internal framework development where deep inspection of the agent's lifecycle is necessary. For most applications, these detailed events should be converted to the simpler [RuntimeStreamEvent](./runtime-stream-event.md) format using the [adaptStream](./adapt-stream.md) function. The simplified format is what standard YAAF runtimes, such as `createCLI` and `createServer`, expect [Source 1].

## Signature

`RunnerStreamEvent` is a discriminated union type. While the exact definition resides in `src/agents/runner.ts`, source material indicates it includes at least the following event types [Source 1]:

- `text_delta`: A chunk of text generated by the LLM.
- `tool_call_start`: Signals the beginning of a tool call.
- `tool_call_result`: Contains the result of a completed tool call.
- `llm_request`: Fired before a request is sent to the LLM, containing request metadata.
- `iteration`: Marks the beginning of a new agent execution loop (e.g., thought -> action -> observation).

The [adaptStream](./adapt-stream.md) function filters out internal-only events like `iteration`, `llm_request`, and `llm_response` when converting to [RuntimeStreamEvent](./runtime-stream-event.md) [Source 1].

```typescript
import type {
  // Hypothetical event detail types
  TextDeltaEvent,
  ToolCallStartEvent,
  ToolCallResultEvent,
  LLMRequestEvent,
  IterationEvent,
} from 'yaaf';

export type RunnerStreamEvent =
  | TextDeltaEvent
  | ToolCallStartEvent
  | ToolCallResultEvent
  | LLMRequestEvent
  | IterationEvent
  // ... and other internal event types
```

## Examples

### Consuming the Raw Event Stream

This example demonstrates how to directly consume the `AsyncIterable<RunnerStreamEvent>` from an agent's `runStream` method and inspect the different internal event types.

```typescript
import { Agent } from 'yaaf';
import type { RunnerStreamEvent } from 'yaaf';

const agent = new Agent({
  systemPrompt: 'You are a helpful assistant that uses tools.',
  // ... other agent configuration
});

async function processRawAgentStream() {
  const stream = agent.runStream('What is the weather in San Francisco?');

  for await (const event of stream) {
    // The 'event' object is of type RunnerStreamEvent
    switch (event.type) {
      case 'text_delta':
        process.stdout.write(event.text);
        break;
      case 'tool_call_start':
        console.log(`\n[INFO] Starting tool call: ${event.toolName}`);
        break;
      case 'tool_call_result':
        console.log(`\n[INFO] Tool call result received.`);
        break;
      case 'iteration':
        console.log(`\n[DEBUG] Starting agent iteration ${event.iteration}`);
        break;
      case 'llm_request':
        console.log(`\n[DEBUG] Sending request to LLM.`);
        break;
      // ... handle other potential internal event types
    }
  }
}

processRawAgentStream();
```
