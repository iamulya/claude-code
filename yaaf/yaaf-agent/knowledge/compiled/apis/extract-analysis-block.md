## Overview

The `extractAnalysisBlock` function is a utility designed to parse the output of a Large Language Model ([LLM](../concepts/llm.md)) and extract the content within an `<analysis>` XML-like block [Source 1].

This function is part of YAAF's structured [Context Compaction](../concepts/context-compaction.md) system. Prompts generated by `buildCompactionPrompt` instruct the model to produce a "scratchpad" of its reasoning inside an `<analysis>` block, followed by the final summary inside a `<summary>` block. `extractAnalysisBlock` provides a way to access this reasoning, which is invaluable for debugging the quality and accuracy of the conversation summary [Source 1].

For safety and performance, the function caps the input text at 512 KB before applying the regular expression. This prevents potential event-loop blocking if the LLM generates an extremely large response without a proper closing `</analysis>` tag [Source 1].

## Signature

```typescript
export function extractAnalysisBlock(text: string): string | null;
```

**Parameters:**

*   `text` (string): The raw string output from the language model, which is expected to contain an `<analysis>...</analysis>` block.

**Returns:**

*   `string | null`: The content of the `<analysis>` block as a string if found. If the block is not present in the input text, it returns `null`.

## Examples

### Example 1: Extracting a valid analysis block

This example demonstrates the primary use case where the model's output contains a well-formed `<analysis>` block.

```typescript
import { extractAnalysisBlock } from 'yaaf';

const modelOutput = `
<analysis>
The user is asking about the project status and seems concerned about the timeline.
Key points to summarize:
1. Project "Phoenix" is delayed by 2 weeks.
2. The cause was an unexpected integration issue.
3. A new delivery date has been set.
</analysis>
<summary>
The team discussed a two-week delay in Project "Phoenix" due to an integration issue and established a new delivery date.
</summary>
`;

const analysisContent = extractAnalysisBlock(modelOutput);

console.log(analysisContent);
/*
Output:

The user is asking about the project status and seems concerned about the timeline.
Key points to summarize:
1. Project "Phoenix" is delayed by 2 weeks.
2. The cause was an unexpected integration issue.
3. A new delivery date has been set.

*/
```

### Example 2: Handling output without an analysis block

If the input string does not contain an `<analysis>` block, the function gracefully returns `null`.

```typescript
import { extractAnalysisBlock } from 'yaaf';

const modelOutputWithoutAnalysis = `
<summary>
The team discussed a two-week delay in Project "Phoenix".
</summary>
`;

const analysisContent = extractAnalysisBlock(modelOutputWithoutAnalysis);

console.log(analysisContent);
// Output: null
```
