## Overview

The `ContradictionReport` interface defines the structure of the object returned by the `detectContradictions` function [Source 1]. It serves as a comprehensive summary of a post-synthesis scan that identifies potential contradictions between compiled knowledge base articles [Source 1].

This report is the output of a heuristic-based detection pass and does not involve an [LLM](../concepts/llm.md). It identifies three main types of contradictions:
- **Negation contradictions**: Sentences with high token overlap where one contains negation.
- **Numeric disagreements**: Conflicting numeric claims about the same entity.
- **Temporal conflicts**: Disagreeing dates or years for the same concept.

The report is for detection only; it highlights potential issues but does not attempt to resolve them. Resolution is left to a human operator or a subsequent automated process [Source 1]. The report also includes metadata about the scan itself, such as the number of articles processed and whether the scan was stopped early due to budget constraints [Source 1].

## Signature

The `ContradictionReport` is a TypeScript interface with the following structure [Source 1]:

```typescript
export interface ContradictionReport {
  pairs: ContradictionPair[];
  articlesScanned: number;
  claimsScanned: number;
  /** True if the scan was stopped early due to maxComparisons budget. */
  truncated: boolean;
}
```

The `pairs` property contains an array of `ContradictionPair` objects, which have the following signature [Source 1]:

```typescript
export interface ContradictionPair {
  articleA: string;
  articleB: string;
  claimA: string;
  claimB: string;
  similarity: number;
  type: "negation" | "numeric_disagreement" | "temporal_conflict";
}
```

## Properties

- **`pairs: ContradictionPair[]`**
  An array of objects, where each object represents a single potential contradiction found between two articles. If no contradictions are found, this array will be empty [Source 1].

- **`articlesScanned: number`**
  The total number of compiled articles that were analyzed during the scan [Source 1].

- **`claimsScanned: number`**
  The total number of individual sentences (claims) that were extracted and compared across all scanned articles [Source 1].

- **`truncated: boolean`**
  A flag that is `true` if the scan was halted before completion because the `maxComparisons` budget defined in `ContradictionOptions` was exceeded. This indicates the report may be incomplete [Source 1].

## Examples

Below is an example of a `ContradictionReport` object that might be returned after a scan. It shows one [Numeric Disagreement](../concepts/numeric-disagreement.md) and indicates that the scan completed fully.

```typescript
const report: ContradictionReport = {
  pairs: [
    {
      articleA: 'src/kb/ProjectTitan.md',
      articleB: 'src/kb/HistoryOfYAAF.md',
      claimA: 'Project Titan was initiated in 2022.',
      claimB: 'The work on Project Titan began in 2023.',
      similarity: 0.85,
      type: 'numeric_disagreement',
    },
  ],
  articlesScanned: 152,
  claimsScanned: 3104,
  truncated: false,
};

if (report.pairs.length > 0) {
  console.log(`Found ${report.pairs.length} potential contradictions.`);
  // Logic to display or log the contradictions for review.
}

if (report.truncated) {
  console.warn('Warning: Contradiction scan was truncated and may be incomplete.');
}
```
