## Overview
The `KBLinter` is the central orchestrator for validating the integrity of a compiled knowledge base (KB). It performs static analysis by scanning compiled articles, running a suite of checks in parallel, and generating a structured report of issues. 

Beyond identification, the `KBLinter` facilitates a "self-healing" workflow. It can automatically resolve specific classes of issues through its fixing engine, reducing the manual overhead required to maintain KB consistency. This process typically follows a three-step pattern:
1.  **Scan**: Analyze the compiled wiki for structural or content issues.
2.  **Auto-fix**: Automatically correct repairable errors.
3.  **Report**: Provide a human-readable summary of remaining issues that require manual intervention.

## Signature / Constructor
The `KBLinter` is initialized with the KB's structural definitions and the location of the compiled artifacts.

```typescript
constructor(
  ontology: KBOntology,
  registry: ConceptRegistry,
  compiledDir: string
)
```

### Parameters
- `ontology`: The `KBOntology` defining the schema and relationships.
- `registry`: The `ConceptRegistry` containing the known entities and their metadata.
- `compiledDir`: The file system path to the directory containing the compiled KB articles.

## Methods & Properties

### lint()
Runs all registered static checks against the compiled articles.
- **Returns**: `Promise<LintReport>` — A structured object containing all detected `LintIssue` entities.

### fix()
Attempts to automatically resolve issues identified in a previous linting pass.
- **Parameters**: 
    - `report`: A `LintReport` generated by the `lint()` method.
- **Returns**: `Promise<AutoFixResult>` — A summary of the changes applied to the KB.

## Examples

### Basic Linting and Fixing
This example demonstrates the standard workflow for identifying and resolving KB issues.

```typescript
import { KBLinter } from 'yaaf/knowledge';

const linter = new KBLinter(ontology, registry, './dist/kb');

// 1. Scan for issues
const report = await linter.lint();
console.log(`Found ${report.issues.length} issues.`);

// 2. Apply automatic fixes
if (report.issues.length > 0) {
  const fixes = await linter.fix(report);
  console.log(`Applied ${fixes.appliedCount} fixes.`);
}
```
