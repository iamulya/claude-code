## Overview
`KnowledgeBaseOptions` is a configuration interface used to initialize or load a `KnowledgeBase` instance. It defines the filesystem structure for the knowledge base, including the root directory and the location of compiled artifacts. It also provides options for namespacing document identifiers, which is critical when integrating multiple knowledge bases into a single agent or using a federated architecture.

## Signature / Constructor
```typescript
export type KnowledgeBaseOptions = {
  /** Path to the KB root directory (contains ontology.yaml, raw/, compiled/) */
  kbDir: string
  /** Name of the compiled directory. Default: 'compiled' */
  compiledDirName?: string
  /** Options for the runtime tools */
  toolOptions?: KBToolOptions
  /**
   * Optional namespace prefix for all docIds returned by tools.
   * When set, docIds become `namespace:docId` (e.g. `ml:concepts/attention`).
   * Recommended when using this KB alongside a FederatedKnowledgeBase,
   * or any context where multiple KBs are in play.
   */
  namespace?: string
}
```

## Methods & Properties
| Property | Type | Description |
| :--- | :--- | :--- |
| `kbDir` | `string` | The absolute or relative path to the Knowledge Base root directory. This directory is expected to contain the `ontology.yaml` file and the `raw/` and `compiled/` subdirectories. |
| `compiledDirName` | `string` | (Optional) The name of the directory containing compiled artifacts. Defaults to `'compiled'`. |
| `toolOptions` | `KBToolOptions` | (Optional) Configuration settings for the tools generated by the Knowledge Base for LLM consumption. |
| `namespace` | `string` | (Optional) A prefix applied to all document IDs (e.g., `namespace:document_id`). This is used to prevent ID collisions when an agent has access to multiple knowledge sources. |

## Examples

### Basic Configuration
Defining a configuration for a local knowledge base with a custom compiled directory.

```typescript
import { KnowledgeBaseOptions } from 'yaaf/knowledge';

const options: KnowledgeBaseOptions = {
  kbDir: './knowledge-assets',
  compiledDirName: 'build-artifacts'
};
```

### Namespaced Configuration
Configuring a knowledge base for use in a federated environment where multiple sources are present.

```typescript
import { KnowledgeBaseOptions } from 'yaaf/knowledge';

const options: KnowledgeBaseOptions = {
  kbDir: './docs/technical-specs',
  namespace: 'tech-specs',
  toolOptions: {
    // tool-specific configurations
  }
};
```
