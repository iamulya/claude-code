## Overview

The `markdown[[[[[[[[Ingester]]]]]]]]` is a constant object that implements the `Ingester` interface for processing Markdown (`.md`) and MDX (`.mdx`) files [Source 1]. It serves as the primary and highest-fidelity ingestion path within the YAAF knowledge subsystem, particularly for content generated by [Tools](../subsystems/tools.md) like the Obsidian Web Clipper or Joplin Web Clipper [Source 1].

Its main responsibilities include:
1.  Reading the raw content of a markdown file.
2.  Parsing YAML [Frontmatter](../concepts/frontmatter.md) to extract metadata.
3.  Determining the document's title, using the frontmatter `title` field or falling back to the first H1 heading.
4.  Resolving all local Markdown image references (e.g., `![](./images/diagram.png)`) to their absolute file paths.
5.  Packaging the extracted text, image paths, and metadata into a normalized `IngestedContent` object [Source 1].

This Ingester has no [Optional Dependencies](../concepts/optional-dependencies.md), ensuring it is always available in a standard YAAF installation [Source 1].

## Signature / Constructor

`markdownIngester` is a constant object, not a class, so it does not have a constructor. It conforms to the `Ingester` interface.

```typescript
import type { Ingester, IngestedContent, [[[[[[[[IngesterOptions]]]]]]]] } from "./types.js";

export const markdownIngester: Ingester = {
  supportedMimeTypes: ["text/markdown"],
  supportedExtensions: ["md", "mdx"],
  requiresOptionalDeps: false,

  ingest(
    filePath: string,
    options?: IngesterOptions
  ): Promise<IngestedContent>;
};
```

### IngesterOptions

The `ingest` method accepts an optional `options` object with the following properties:

*   `imageOutputDir` (`string`, optional): The directory where resolved images should be placed. Defaults to an `assets` subdirectory within the same directory as the source markdown file (e.g., `join(dirname(filePath), "assets")`) [Source 1].
*   `sourceUrl` (`string`, optional): The original source URL of the content, if available. This takes precedence over any URL found in the frontmatter [Source 1].

## Methods & Properties

### Properties

*   **`supportedMimeTypes`**: `string[]`
    An array of MIME types this ingester can handle. Value: `["text/markdown"]` [Source 1].

*   **`supportedExtensions`**: `string[]`
    An array of file extensions this ingester supports. Value: `["md", "mdx"]` [Source 1].

*   **`requiresOptionalDeps`**: `boolean`
    Indicates whether the ingester requires optional dependencies to be installed. Value: `false` [Source 1].

### Methods

*   **`ingest(filePath, options)`**
    Asynchronously processes a markdown file and returns a structured `IngestedContent` object.

    **Parameters:**
    *   `filePath`: `string` - The absolute path to the `.md` or `.mdx` file to ingest.
    *   `options`: `IngesterOptions` (optional) - Configuration for the ingestion process.

    **Returns:** `Promise<IngestedContent>`
    A promise that resolves to an `IngestedContent` object with the following structure:
    *   `text`: The body of the markdown file (content without frontmatter).
    *   `images`: An array of absolute paths to all resolved images found in the document.
    *   `mimeType`: The string `"text/markdown"`.
    *   `sourceFile`: The original `filePath` provided as input.
    *   `title`: The extracted document title.
    *   `metadata`: An object containing all key-value pairs from the YAML frontmatter. If any images could not be resolved, this object will also contain a `_unresolved_images` key with an array of the unresolved paths.
    *   `lossy`: A boolean indicating if the ingestion process lost information. For this ingester, it is always `false`.
    *   `sourceUrl`: The original URL of the content, taken from `options.sourceUrl` or the `source` or `url` keys in the frontmatter.

## Examples

### Basic Ingestion

Consider a markdown file `notes/research-paper.md` with the following content:

```markdown
---
author: "Dr. Eva Core"
source: "https://example.com/research"
tags: ["ai", "agents"]
---

# A Study on Agent Frameworks

This document outlines the key findings.

See the architecture diagram below:
![Architecture Diagram](./assets/diagram.png)
```

The ingester would process this file and produce an `IngestedContent` object similar to this:

```typescript
// Conceptual usage within the knowledge compiler
const filePath = "/path/to/your/project/notes/research-paper.md";
const ingestedContent = await markdownIngester.ingest(filePath);

/*
ingestedContent would be:
{
  text: '\n# A Study on Agent Frameworks\n\nThis document outlines the key findings.\n\nSee the architecture diagram below:\n![Architecture Diagram](./assets/diagram.png)',
  images: [ '/path/to/your/project/notes/assets/diagram.png' ],
  mimeType: 'text/markdown',
  sourceFile: '/path/to/your/project/notes/research-paper.md',
  title: 'A Study on Agent Frameworks',
  metadata: {
    author: 'Dr. Eva Core',
    source: 'https://example.com/research',
    tags: [ 'ai', 'agents' ]
  },
  lossy: false,
  sourceUrl: 'https://example.com/research'
}
*/
```
