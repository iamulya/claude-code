## Overview

The `GroundingSentence` type is a data structure that holds the detailed results of a grounding validation check for a single sentence from an [LLM](../concepts/llm.md)'s response [Source 1]. It is a key component of the `GroundingAssessment` object, which is generated by the `GroundingValidator` during its analysis.

Each `GroundingSentence` object provides information on whether a specific sentence is considered "grounded" in the provided evidence (such as [tool results](../concepts/tool-results.md)), the method used to make that determination, and quantitative details like the number of overlapping keywords [Source 1]. This per-sentence breakdown allows for fine-grained analysis of an LLM's tendency to hallucinate or make claims not supported by facts.

## Signature

`GroundingSentence` is a TypeScript type with the following structure [Source 1]:

```typescript
export type GroundingSentence = {
  /** The sentence text */
  text: string;
  /** Whether this sentence is grounded in tool evidence */
  grounded: boolean;
  /** Number of overlapping tokens with tool evidence */
  overlapCount: number;
  /** Which tool result provided the best evidence (if any) */
  bestSource?: string;
  /**
   * Which method determined grounding.
   * 'keyword' = [[[[[[[[TF-IDF]]]]]]]] overlap; 'embedding' = [[[[[[[[Cosine Similarity]]]]]]]]; 'llm' = LLM scorer.
   */
  scoredBy?: "keyword" | "embedding" | "llm";
};
```

### Properties

*   **`text`**: `string`
    The original text of the sentence that was evaluated.

*   **`grounded`**: `boolean`
    The final assessment of the sentence. `true` if the sentence is considered supported by the evidence, `false` otherwise.

*   **`overlapCount`**: `number`
    The number of significant keywords that overlap between the sentence and the best-matching piece of evidence.

*   **`bestSource`**: `string` (optional)
    The content of the tool result or evidence snippet that provided the strongest match for this sentence. This field is present if a grounding source was found.

*   **`scoredBy`**: `"keyword" | "embedding" | "llm"` (optional)
    Indicates which scoring mechanism made the final determination for the `grounded` status.
    *   `'keyword'`: Grounding was determined by TF-IDF keyword overlap.
    *   `'embedding'`: Grounding was determined by Cosine Similarity between [Vector Embeddings](../concepts/vector-embeddings.md).
    *   `'llm'`: Grounding was determined by a call to an external LLM scorer.

## Examples

A `GroundingSentence` object is typically found within the `sentences` array of a `GroundingAssessment` object. The following example shows a possible assessment for an LLM response containing two sentences.

```typescript
import type { GroundingAssessment } from 'yaaf';

const assessment: GroundingAssessment = {
  score: 0.5,
  totalSentences: 2,
  groundedSentences: 1,
  action: "warned",
  timestamp: new Date(),
  sentences: [
    {
      text: "The current weather in London is 15°C and partly cloudy.",
      grounded: true,
      overlapCount: 5,
      bestSource: "getWeather(city='London') -> {'temperature': '15°C', 'conditions': 'partly cloudy'}",
      scoredBy: "keyword"
    },
    {
      text: "It is expected to rain later this evening.",
      grounded: false,
      overlapCount: 0,
      // bestSource is undefined because no evidence was found.
    }
  ]
};

console.log(assessment.sentences[0].text);
// > "The current weather in London is 15°C and partly cloudy."
console.log(assessment.sentences[0].grounded);
// > true

console.log(assessment.sentences[1].text);
// > "It is expected to rain later this evening."
console.log(assessment.sentences[1].grounded);
// > false
```
