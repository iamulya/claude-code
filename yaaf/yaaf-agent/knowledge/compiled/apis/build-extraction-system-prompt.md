## Overview

The `buildExtractionSystemPrompt` function constructs the [System Prompt](../concepts/system-prompt.md) used in the [LLM](../concepts/llm.md)-based knowledge extraction pass [Source 1]. This is the first of two prompts sent to the extraction model; the second is the user prompt generated by `buildExtractionUserPrompt`.

The primary purpose of this system prompt is to ground the LLM in the domain of the target knowledge base before it processes the source content. It provides the model with the necessary context and constraints to generate a valid [JSON Compilation Plan](../concepts/json-compilation-plan.md) [Source 1].

Specifically, the generated prompt includes:
- The [Ontology](../concepts/ontology.md)'s entity types and their [Frontmatter](../concepts/frontmatter.md) schemas.
- A minimal and explicit JSON schema for the expected output.
- Few-shot examples demonstrating the correct JSON format [Source 1].

This approach follows [Prompt Engineering](../concepts/prompt-engineering.md) principles by front-loading the most constraining information (the ontology) to guide the model's behavior effectively [Source 1].

## Signature

```typescript
export function buildExtractionSystemPrompt(ontology: KBOntology): string;
```

### Parameters

- **`ontology`** `KBOntology`: An object representing the knowledge base's ontology, including all defined entity types and their schemas [Source 1].

### Returns

- `string`: A formatted string to be used as the system prompt for the extraction LLM [Source 1].

## Examples

The following example demonstrates how to build a system prompt from a simple [Knowledge Base Ontology](../subsystems/knowledge-base-ontology.md).

```typescript
import { buildExtractionSystemPrompt } from 'yaaf';
import type { KBOntology } from 'yaaf';

// A simplified representation of a knowledge base ontology
const myOntology: KBOntology = {
  entityTypes: {
    api: {
      name: 'api',
      description: 'A function, class, or type in the public API.',
      frontmatterSchema: {
        title: { type: 'string', required: true },
        category: { type: 'string', required: true },
      },
    },
    concept: {
      name: 'concept',
      description: 'An abstract idea or high-level concept.',
      frontmatterSchema: {
        title: { type: 'string', required: true },
      },
    },
  },
  // In a real scenario, other ontology properties would be present
};

const systemPrompt = buildExtractionSystemPrompt(myOntology);

console.log(systemPrompt);
/*
Output will be a detailed string instructing an LLM on how to act as a
knowledge extractor. It will define the available entity types ('api', 'concept'),
their schemas, and the required JSON output format for the compilation plan.
*/
```
