## Overview

`YAAF[[[[[[[[Span]]]]]]]]Type` is a TypeScript string literal type that enumerates the distinct categories of operations Traced by the YAAF framework's internal [Telemetry System](../subsystems/telemetry-system.md) [Source 1]. Each value represents a specific, logical unit of work within an agent's lifecycle.

This type is a core component of YAAF's [Observability](../concepts/observability.md) features. [when](./when.md) the framework creates an [OpenTelemetry](../concepts/open-telemetry.md) Span, it assigns one of these types. This categorization is crucial for consumers of telemetry data, as it allows for precise filtering, aggregation, and analysis of Traces, making it easier to debug performance issues or understand agent behavior. For example, one could easily isolate all [LLM](../concepts/llm.md) requests or all blocked [Tool Calls](../concepts/tool-calls.md) within a [Trace](../concepts/trace.md) [Source 1].

## Signature

`YAAFSpanType` is defined as a union of the following string literals [Source 1]:

```typescript
export type YAAFSpanType =
  | "agent.run"
  | "llm.request"
  | "tool.call"
  | "tool.execution"
  | "tool.blocked"
  | "memory.extract"
  | "memory.retrieve"
  | "compaction";
```

### Span Types

| Type               | Description                                                              |
| ------------------ | ------------------------------------------------------------------------ |
| `agent.run`        | Represents a single, complete invocation of the `Agent.run()` method.    |
| `llm.request`      | Corresponds to one call to a chat model's completion method, e.g., `ChatModel.complete()`. |
| `tool.call`        | Traces a tool invocation before any security checks like [Sandboxing](../subsystems/sandboxing.md) or permission policies are applied. |
| `tool.execution`   | Traces the actual execution of the tool's function, typically inside a secure sandbox. |
| `tool.blocked`     | Indicates an attempted tool call that was denied by a permission policy or a pre-execution hook. |
| `memory.extract`   | Represents a call to a `MemoryStrategy.extract()` method.                |
| `memory.retrieve`  | Represents a call to a `MemoryStrategy.retrieve()` method.               |
| `compaction`       | Denotes the execution of a [Context Compaction](../concepts/context-compaction.md) strategy.                  |

## Examples

The `YAAFSpanType` is used internally by YAAF's telemetry functions, such as `buildSpanAttributes`, to construct the appropriate metadata for a new trace span.

```typescript
import { buildSpanAttributes, YAAFSpanType } from 'yaaf';
import type { Attributes } from '@opentelemetry/api';

// This function is a conceptual example of how YAAFSpanType is used internally.
function createTraceSpan(spanType: YAAFSpanType, customAttributes: Attributes) {
  // The spanType is passed to a utility to build a full attribute set.
  const attributes = buildSpanAttributes(spanType, customAttributes);
  
  console.log(`Starting a new span with attributes:`, attributes);
  // In a real scenario, this would use an OpenTelemetry tracer to start a span.
  // tracer.startSpan('span-name', { attributes });
}

// Example usage for an LLM request span
createTraceSpan('llm.request', {
  'llm.model_name': 'gpt-4o',
  'llm.provider': 'openai',
});

// Example usage for a blocked tool call span
createTraceSpan('tool.blocked', {
  'tool.name': 'execute_code',
  'tool.block_reason': 'permission_denied',
});
```
