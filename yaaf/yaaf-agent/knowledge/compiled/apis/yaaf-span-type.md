## Overview
`YAAFSpanType` is a TypeScript union type that categorizes the various operations tracked by the YAAF telemetry system. It is used to ensure consistent naming conventions and cardinality dimensions across OpenTelemetry spans generated by the framework's tracing module. Every span created within YAAF is assigned one of these types to facilitate standardized observability and filtering in distributed tracing backends.

## Signature
```typescript
export type YAAFSpanType =
  | 'agent.run'         // one call to Agent.run()
  | 'llm.request'       // one ChatModel.complete() call
  | 'tool.call'         // one tool invocation (before sandbox/permissions)
  | 'tool.execution'    // the tool fn itself inside the sandbox
  | 'tool.blocked'      // tool denied by permission policy or hook
  | 'memory.extract'    // MemoryStrategy.extract()
  | 'memory.retrieve'   // MemoryStrategy.retrieve()
  | 'compaction';       // context compaction strategy run
```

## Methods & Properties
As a union type, `YAAFSpanType` consists of the following literal string values:

| Value | Description |
| :--- | :--- |
| `agent.run` | Represents a single execution cycle of an agent's primary run method. |
| `llm.request` | Represents an outgoing request to a Large Language Model provider. |
| `tool.call` | Represents the initial attempt to invoke a tool, occurring before sandbox entry or permission validation. |
| `tool.execution` | Represents the actual execution of a tool's logic within its designated sandbox. |
| `tool.blocked` | Represents a tool invocation that was prevented by a permission policy or a lifecycle hook. |
| `memory.extract` | Represents the operation of extracting and storing information via a memory strategy. |
| `memory.retrieve` | Represents the operation of fetching relevant information from a memory strategy. |
| `compaction` | Represents the execution of a strategy designed to reduce or summarize the conversation context. |

## Examples

### Using YAAFSpanType in a custom tracing function
This example demonstrates how to use the type to ensure a function only accepts valid YAAF span categories.

```typescript
import { YAAFSpanType } from './telemetry/attributes';

function logSpanStart(type: YAAFSpanType) {
  console.log(`Starting telemetry span for: ${type}`);
}

// Valid usage
logSpanStart('agent.run');
logSpanStart('llm.request');

// @ts-expect-error - 'custom.operation' is not a valid YAAFSpanType
logSpanStart('custom.operation');
```

### Integration with buildSpanAttributes
The framework uses `YAAFSpanType` in utility functions to construct standardized attribute sets for OpenTelemetry.

```typescript
import { YAAFSpanType, buildSpanAttributes } from './telemetry/attributes';

const attributes = buildSpanAttributes('tool.execution', {
  'tool.name': 'calculator'
});
```
