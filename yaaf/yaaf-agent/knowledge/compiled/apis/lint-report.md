## Overview

The `LintReport` type represents the complete output of a knowledge base (KB) [Linting](../concepts/linting.md) process. It is a data structure that aggregates all detected issues, along with summary statistics about the linting run.

This type is the primary return value from the [Linter](../concepts/linter.md) subsystem. Developers typically inspect an object of this type to programmatically check for errors, generate human-readable reports, or fail a build process if critical issues are found. It serves as a comprehensive snapshot of the KB's structural and semantic health at a given point in time.

## Signature

`LintReport` is exported as a TypeScript `type` from the linter's public API. While its detailed structure is defined in `src/knowledge/compiler/linter/types.ts`, the provided source material only shows its export [Source 1].

Conceptually, a `LintReport` object contains a list of all `LintIssue` objects found, as well as summary metadata such as the number of files scanned and counts of issues categorized by severity.

## Examples

The most common use case for a `LintReport` is to process the results of a linting run. The following example shows a function that takes a `LintReport` and prints a summary to the console, exiting with an error code if any errors were found.

```typescript
import { LintReport, LintSeverity } from 'yaaf';

/**
 * A hypothetical function that processes a lint report.
 * Note: The exact properties of LintReport are illustrative.
 */
function processLintReport(report: LintReport): void {
  const { stats, issues, summary } = report;

  console.log(`\nLinting complete. Scanned ${stats.filesScanned} files.`);

  if (issues.length === 0) {
    console.log('✅ No issues found.');
    return;
  }

  const errorCount = summary.bySeverity[LintSeverity.Error] ?? 0;
  const warningCount = summary.bySeverity[LintSeverity.Warning] ?? 0;

  console.log(
    `Found ${issues.length} total issues: ${errorCount} error(s), ${warningCount} warning(s).`
  );

  // Log details for each issue
  for (const issue of issues) {
    console.log(
      `[${issue.severity.toUpperCase()}] ${issue.code}: ${issue.message} (${issue.location.filePath})`
    );
  }

  if (errorCount > 0) {
    console.error('\n❌ Build failed due to linting errors.');
    process.exit(1);
  }
}
```
