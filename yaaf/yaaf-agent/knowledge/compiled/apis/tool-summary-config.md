## Overview

`ToolSummaryConfig` is a TypeScript type that defines the configuration object required by the `generateToolUseSummary` function [Source 1]. This function is used to generate a concise, human-readable, one-line summary of a batch of [Tool Execution](../concepts/tool-execution.md)s, such as "Fixed NPE in UserService" or "Read config.json" [Source 1].

The `ToolSummaryConfig` object provides all the necessary information for a language model to perform this summarization, including the list of [Tools](../subsystems/tools.md) that were run, the model to use for generation, and optional contextual information [Source 1].

## Signature

`ToolSummaryConfig` is a type alias for an object with the following properties. It relies on the `ToolInfo` type to describe each executed tool [Source 1].

```typescript
import type { ChatModel } from "../agents/runner.js";

export type ToolInfo = {
  name: string;
  input: unknown;
  output: unknown;
};

export type ToolSummaryConfig = {
  /** Tools executed in this batch. */
  tools: ToolInfo[];

  /** The model to use for summarization (small/fast recommended). */
  model: ChatModel;

  /** Abort signal. */
  signal?: AbortSignal;

  /** Most recent assistant text for context. */
  lastAssistantText?: string;
};
```

### Properties

*   **`tools`**: `ToolInfo[]` (required)
    An array of `ToolInfo` objects, where each object represents a single Tool Execution in the batch. A `ToolInfo` object contains the tool's `name`, its `input`, and its `output` [Source 1].

*   **`model`**: `ChatModel` (required)
    The language model instance to be used for generating the summary. The documentation recommends using a small and [Fast Model](../concepts/fast-model.md) for this task [Source 1].

*   **`signal`**: `AbortSignal` (optional)
    An optional `AbortSignal` that can be used to cancel the summary generation process [Source 1].

*   **`lastAssistantText`**: `string` (optional)
    The most recent text generated by the assistant. This can provide valuable context to the model for creating a more relevant summary [Source 1].

## Examples

The following example demonstrates how to create a `ToolSummaryConfig` object and pass it to the `generateToolUseSummary` function to get a summary of file operations.

```typescript
import { generateToolUseSummary, ToolSummaryConfig } from 'yaaf';
import { smallModel } from './my-models'; // Assume a model is imported

async function summarizeFileEdits() {
  // Configuration object for the summary generator
  const config: ToolSummaryConfig = {
    tools: [
      {
        name: 'read_file',
        input: { path: 'src/auth.ts' },
        output: '...' // file content
      },
      {
        name: 'edit_file',
        input: { path: 'src/auth.ts', changes: '...' },
        output: 'OK'
      },
    ],
    model: smallModel,
    lastAssistantText: "Okay, I see the validation bug. I will read the file and apply a fix."
  };

  const summary = await generateToolUseSummary(config);

  console.log(summary);
  // Expected output: "Fixed auth validation in auth.ts"
}
```
