## Overview
`RuntimeStreamEvent` is a discriminated union type that represents simplified events emitted during an agent's execution. It serves as a bridge between the detailed internal events produced by the agent runner and the high-level requirements of runtime harnesses such as CLIs, servers, or workers.

While the internal runner yields granular data—including iteration counts and raw LLM request metadata—`RuntimeStreamEvent` strips these details to provide a clean, consumer-facing interface focused on text streaming, tool lifecycle, and final usage statistics.

## Signature / Constructor

```typescript
export type RuntimeStreamEvent =
  | { type: 'text_delta'; text: string }
  | { type: 'tool_call_start'; toolName: string; args?: any }
  | { type: 'tool_call_end'; toolName: string; durationMs?: number; error?: any }
  | { type: 'tool_blocked'; toolName: string; reason: string }
  | { type: 'usage'; promptTokens: number; completionTokens: number; totalCalls: number }
  | { type: 'done'; text: string };
```

## Event Types

The union consists of the following event objects:

| Event Type | Fields | Description |
|:---|:---|:---|
| `text_delta` | `text: string` | A token chunk or text fragment generated by the LLM. |
| `tool_call_start` | `toolName: string`, `args?: any` | Emitted when a tool execution begins. |
| `tool_call_end` | `toolName: string`, `durationMs?: number`, `error?: any` | Emitted when a tool execution completes, optionally including timing and error data. |
| `tool_blocked` | `toolName: string`, `reason: string` | Emitted when a tool execution is denied (e.g., by a security policy or user intervention). |
| `usage` | `promptTokens: number`, `completionTokens: number`, `totalCalls: number` | Provides final or intermediate token usage and tool call counts. |
| `done` | `text: string` | Signifies the end of the stream and contains the final, complete response text. |

## Examples

### Adapting an Agent Stream
The `adaptStream` function can be used to convert internal runner events into `RuntimeStreamEvent` objects for manual consumption.

```typescript
import { Agent, adaptStream } from 'yaaf';

const agent = new Agent({ /* ... */ });

for await (const event of adaptStream(agent.runStream('Hello'))) {
  if (event.type === 'text_delta') {
    process.stdout.write(event.text);
  } else if (event.type === 'tool_call_start') {
    console.log(`\nExecuting ${event.toolName}...`);
  }
}
```

### Usage with Runtime Harnesses
Runtimes like `createCLI` expect a `StreamableAgent` that yields `RuntimeStreamEvent`. The `toStreamableAgent` utility handles this conversion automatically.

```typescript
import { Agent, toStreamableAgent } from 'yaaf';
import { createCLI } from 'yaaf/cli-runtime';

const agent = new Agent({ ... });

// createCLI consumes RuntimeStreamEvents internally to update the UI
createCLI(toStreamableAgent(agent), {
  streaming: true,
  name: 'assistant'
});
```
