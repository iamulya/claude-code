## Overview
The `validateFrontmatter` function is a core utility in the YAAF knowledge compiler's synthesis pipeline. It ensures that metadata generated by a Large Language Model (LLM) conforms to the structural and type constraints defined in the project's ontology.

The function performs strict validation for required fields while remaining lenient for optional ones to prevent minor inconsistencies from blocking the compilation process. It includes built-in coercion logic to handle common LLM output patterns, such as strings representing booleans or numbers.

## Signature
```typescript
export function validateFrontmatter(
  rawValues: Record<string, unknown>,
  schema: FrontmatterSchema,
  entityType: string,
  registry: ConceptRegistry,
  ontology: KBOntology,
): FrontmatterValidationResult
```

### Parameters
- `rawValues`: A key-value map of the unvalidated frontmatter fields extracted from the LLM output.
- `schema`: The `FrontmatterSchema` defining the expected fields and types for the specific entity.
- `entityType`: The string identifier for the entity type being validated.
- `registry`: The `ConceptRegistry` used to verify entity references.
- `ontology`: The `KBOntology` containing the global schema definitions.

### Validation Rules
The function applies specific logic based on the field type defined in the schema:
- **string / url**: Coerces the value to a string.
- **string[] / url[]**: Coerces the value to an array of strings.
- **number**: Attempts to parse a number from a string if the raw value is not already a number.
- **boolean**: Coerces the strings `'true'` and `'false'` to their respective boolean values.
- **date**: Accepts the value as a string but issues a warning if it does not follow the ISO 8601 format.
- **enum / enum[]**: Validates that the provided value (or values) exists within the schema's allowed list.
- **entity_ref / entity_ref[]**: Checks the `ConceptRegistry` to ensure the referenced `docId` exists.

## Examples

### Validating LLM Output
This example demonstrates how the function processes raw LLM output against a schema, including type coercion for a boolean field.

```typescript
const rawLLMOutput = {
  title: "Agent Framework",
  is_production_ready: "true",
  category: "framework"
};

const schema = {
  fields: {
    title: { type: "string", required: true },
    is_production_ready: { type: "boolean", required: true },
    category: { type: "enum", options: ["framework", "library"], required: true }
  }
};

const result = validateFrontmatter(
  rawLLMOutput,
  schema,
  "concept",
  conceptRegistry,
  kbOntology
);

if (result.valid) {
  console.log(result.values.is_production_ready); // true (boolean)
} else {
  console.error(result.errors);
}
```
