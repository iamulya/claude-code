## Overview

The `validate[[[[[[[[Frontmatter]]]]]]]]` function is a core component of the YAAF [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md). Its primary role is to take raw, unstructured key-value data, typically generated by a Large Language Model ([LLM](../concepts/llm.md)), and validate it against a formal `FrontmatterSchema` defined in the project's [Ontology](../concepts/ontology.md) [Source 1].

This process ensures that all compiled articles have structurally correct and consistent metadata. The validation is designed to be strict for required fields to prevent compilation with missing critical information, while remaining lenient for optional fields to avoid blocking compilation due to minor inconsistencies [Source 1].

The function performs type coercion and validation based on the field type defined in the schema [Source 1]:
- **string / url**: Coerces the input value to a string.
- **string[] / url[]**: Coerces the input to an array of strings.
- **number**: Attempts to parse a number from a string value.
- **boolean**: Coerces string values of `'true'` or `'false'` to their boolean equivalents.
- **date**: Accepts the value as a string but will issue a warning if it is not in ISO format.
- **enum / enum[]**: Checks if the value (or values) exists in the predefined list of allowed options for that enum.
- **entity_ref / entity_ref[]**: Verifies that the document ID(s) exist in the provided `ConceptRegistry`.

## Signature

```typescript
export function validateFrontmatter(
  rawValues: Record<string, unknown>,
  schema: FrontmatterSchema,
  entityType: string,
  registry: ConceptRegistry,
  ontology: KBOntology,
): FrontmatterValidationResult;
```

**Parameters:**

- `rawValues: Record<string, unknown>`: An object containing the key-value pairs of Frontmatter to be validated, typically parsed from an LLM's output.
- `schema: FrontmatterSchema`: The schema definition from the ontology that corresponds to the `entityType`. It defines the expected fields, their types, and whether they are required.
- `entityType: string`: The type of the entity being processed, such as `'api'` or `'concept'`. This is used to look up the correct schema.
- `registry: ConceptRegistry`: A registry of all known entities in the knowledge base, used to validate `entity_ref` fields.
- `ontology: KBOntology`: The complete ontology instance containing all schema definitions.

**Returns:**

- `FrontmatterValidationResult`: An object that contains the results of the validation, including the coerced and validated values, and a list of any errors or warnings encountered.

## Examples

The following example demonstrates how `validateFrontmatter` is used to validate a set of raw values against a schema.

```typescript
import { validateFrontmatter } from './frontmatter';
import type { FrontmatterSchema, ConceptRegistry, KBOntology } from './types';
import { FieldType } from './types';

// 1. Define a mock schema for an 'api' entity type
const mockApiSchema: FrontmatterSchema = {
  fields: {
    title: { type: FieldType.String, required: true },
    category: { type: FieldType.Enum, required: true, allowedValues: ['class', 'function', 'type'] },
    deprecated: { type: FieldType.Boolean, required: false },
    related_apis: { type: FieldType.EntityRefArray, required: false },
  }
};

// 2. Mock the ontology and concept registry
const mockOntology: KBOntology = { /* ... ontology implementation ... */ };
const mockRegistry: ConceptRegistry = {
  has: (docId: string) => ['serializeFrontmatter'].includes(docId),
  // ... other registry methods
};

// 3. Provide raw frontmatter values from an LLM
const rawFrontmatter = {
  title: 'validateFrontmatter',
  category: 'function', // Correct enum value
  deprecated: 'false', // String that will be coerced to boolean
  related_apis: ['serializeFrontmatter', 'nonExistentApi'], // One valid, one invalid ref
  extra_field: 'This will be ignored'
};

// 4. Call the validator
const result = validateFrontmatter(
  rawFrontmatter,
  mockApiSchema,
  'api',
  mockRegistry,
  mockOntology
);

/*
Expected `result` structure:

result.validatedValues would be:
{
  title: 'validateFrontmatter',
  category: 'function',
  deprecated: false,
  related_apis: ['serializeFrontmatter'] // nonExistentApi is filtered out
}

result.errors would contain an error for the invalid entity_ref 'nonExistentApi'.
result.warnings would be empty in this case.
*/
```
