## Overview

The `StructuredLogBackend` interface defines a minimal contract for a structured logging provider within the YAAF framework [Source 1]. It allows developers to replace the default logging mechanism with a custom implementation, such as the popular `pino` logger or any other library that conforms to the interface.

This enables flexible integration with existing logging infrastructure and allows logs generated by the YAAF `Logger` to be directed to various destinations (e.g., files, cloud logging services, or [Observability](../concepts/observability.md) platforms) in a structured format like NDJSON.

An object that implements `StructuredLogBackend` can be registered with the framework's static `Logger` class via the `Logger.setBackend()` method [Source 1].

## Signature

`StructuredLogBackend` is a TypeScript interface with the following definition [Source 1]:

```typescript
export interface StructuredLogBackend {
  debug(obj: Record<string, unknown>, msg: string): void;
  info(obj: Record<string, unknown>, msg: string): void;
  warn(obj: Record<string, unknown>, msg: string): void;
  error(obj: Record<string, unknown>, msg: string): void;
  child(bindings: Record<string, unknown>): StructuredLogBackend;
}
```

The interface defines five required methods [Source 1]:

### `debug()`

- **`obj`**: A record containing structured data to be included in the log entry.
- **`msg`**: The log message string.

### `child()`
Creates a new logger instance that includes a set of predefined bindings in every log message it generates. This is useful for creating contextual loggers (e.g., for a specific request or agent).

**Signature:**
```typescript
child(bindings: Record<string, unknown>): StructuredLogBackend;
```
- **`bindings`**: A record of key-value pairs to be permanently attached to the new child logger.
- **Returns**: A new instance of a `StructuredLogBackend` with the specified bindings.

## Examples

### Implementing a Custom Console Logger

This example shows how to create a simple, custom logging backend that implements `StructuredLogBackend` and writes structured JSON to the console. This custom backend is then registered with the YAAF `Logger`.

```typescript
import { Logger, StructuredLogBackend } from 'yaaf';

// 1. Create a custom logger class that implements the interface
class CustomJsonConsoleLogger implements StructuredLogBackend {
  private bindings: Record<string, unknown>;

  constructor(bindings: Record<string, unknown> = {}) {
    this.bindings = bindings;
  }

  private log(level: string, obj: Record<string, unknown>, msg: string) {
    const logEntry = {
      level,
      msg,
      ...this.bindings,
      ...obj,
      timestamp: new Date().toISOString(),
    };
    console.log(JSON.stringify(logEntry));
  }

  debug(obj: Record<string, unknown>, msg: string): void {
    this.log('debug', obj, msg);
  }
  info(obj: Record<string, unknown>, msg: string): void {
    this.log('info', obj, msg);
  }
  warn(obj: Record<string, unknown>, msg: string): void {
    this.log('warn', obj, msg);
  }
  error(obj: Record<string, unknown>, msg: string): void {
    this.log('error', obj, msg);
  }
  child(bindings: Record<string, unknown>): StructuredLogBackend {
    const newBindings = { ...this.bindings, ...bindings };
    return new CustomJsonConsoleLogger(newBindings);
  }
}

// 2. Instantiate the custom logger and set it as the global backend
const myLoggerBackend = new CustomJsonConsoleLogger();
Logger.setBackend(myLoggerBackend);

// 3. Use the YAAF Logger as usual throughout the application
const agentLog = new Logger('MyAgent');
agentLog.info('Agent is starting up', { agentId: 'agent-123' });

// Expected console output (timestamp will vary):
// {"level":"info","msg":"Agent is starting up","agentId":"agent-123","name":"MyAgent","timestamp":"2023-10-27T10:00:00.000Z"}
```
