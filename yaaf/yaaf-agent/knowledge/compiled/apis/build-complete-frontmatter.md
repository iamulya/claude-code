## Overview

The `buildComplete[[[[[[[[Frontmatter]]]]]]]]` function is a key part of the knowledge base compilation process. It assembles the final, complete Frontmatter for a compiled article by merging data from three distinct sources, each with a different priority [Source 1].

This function ensures that automatically generated or suggested metadata is combined logically, with compiler-enforced fields taking precedence to maintain structural integrity and consistency across the knowledge base [Source 1].

The merge priority is as follows [Source 1]:
1.  **[LLM](../concepts/llm.md)-generated and validated values**: The primary source of content-specific metadata, such as `title` and `summary`.
2.  **Compiler-suggested frontmatter**: Values inferred by the compiler's `ExtractorPlan` during the source analysis phase. These are used as fallbacks if not provided by the LLM.
3.  **[Compiler-injected Metadata](../concepts/compiler-injected-metadata.md)**: System-level metadata like `entity_type`, `compiled_at`, and source file information. These values always overwrite any conflicting keys from the other sources to ensure consistency.

## Signature

```typescript
export function buildCompleteFrontmatter(
  validatedValues: Record<string, unknown>,
  suggestedFrontmatter: Record<string, unknown>,
  compilerMeta: {
    entity_type: string;
    stub?: boolean;
    compiled_at: string; // ISO timestamp
    compiled_from: string[];
    compiled_from_quality?: string;
    confidence?: number;
    // ... and other potential compiler-injected fields
  }
): Record<string, unknown>;
```

### Parameters

| Name                 | Type                        | Description                                                                                                                                                           |
| -------------------- | --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `validatedValues`    | `Record<string, unknown>`   | An object containing frontmatter fields generated by the LLM and subsequently validated against the Knowledge Base [Ontology](../concepts/ontology.md) [Source 1].                                   |
| `suggestedFrontmatter` | `Record<string, unknown>`   | An object containing frontmatter fields suggested by the compiler's `ExtractorPlan`. These serve as default or fallback values [Source 1].                            |
| `compilerMeta`       | `object`                    | An object containing metadata injected directly by the compiler. These fields are considered authoritative and will overwrite any other values for the same keys [Source 1]. |

### Returns

Returns a single `Record<string, unknown>` object representing the final, merged frontmatter for the article.

## Examples

The following example demonstrates the merge priority. Note how `validatedValues` overrides `suggestedFrontmatter` for the `title`, and `compilerMeta` overrides both for `entity_type`.

```typescript
import { buildCompleteFrontmatter } from 'yaaf/knowledge';

// 1. Frontmatter from the LLM, after validation
const validatedValues = {
  title: "Agent Architecture",
  summary: "An overview of the core components of a YAAF agent.",
  entity_type: "guide", // This will be overridden by compilerMeta
  search_terms: ["agent components", "system design"]
};

// 2. Frontmatter suggested by the compiler's ExtractorPlan
const suggestedFrontmatter = {
  title: "Architecture", // Will be overridden by validatedValues
  category: "Core Concepts"
};

// 3. Metadata injected by the compiler
const compilerMeta = {
  entity_type: "concept", // Overrides the value from validatedValues
  compiled_at: "2023-10-27T10:00:00Z",
  compiled_from: ["src/agent.ts"]
};

const finalFrontmatter = buildCompleteFrontmatter(
  validatedValues,
  suggestedFrontmatter,
  compilerMeta
);

console.log(finalFrontmatter);
/*
Output:
{
  title: "Agent Architecture",
  summary: "An overview of the core components of a YAAF agent.",
  search_terms: ["agent components", "system design"],
  category: "Core Concepts",
  entity_type: "concept",
  compiled_at: "2023-10-27T10:00:00Z",
  compiled_from: ["src/agent.ts"]
}
*/
```
