## Overview
`FederatedKnowledgeBase` is a utility class designed for cross-KB federation. It aggregates multiple `KnowledgeBase` instances into a single interface, organizing them under unique namespaces. This allows an agent to perform unified searches, browse a merged index, and retrieve documents across disparate knowledge sources.

The class enforces specific namespacing rules:
- All document IDs (`docId`) are prefixed with their namespace (e.g., `ml:concepts/attention`).
- Search results and index entries are tagged with their source namespace.
- Tools generated by the federated instance (such as `search_kb` and `fetch_kb_document`) handle namespaced identifiers automatically.

## Methods & Properties

### `tools()`
Returns an array of tools (e.g., `search_kb`, `fetch_kb_document`) configured to operate across all federated knowledge bases. These tools handle the resolution of namespaced document IDs.

### `systemPromptSection()`
Generates a formatted string intended for inclusion in an agent's system prompt. This section describes the available knowledge namespaces and their contents to help the LLM understand which source to query.

### Supporting Types
The class utilizes several namespaced types for its operations:
- **NamespacedDocument**: Extends the standard document with `namespace` and `qualifiedId`.
- **NamespacedSearchResult**: Extends search results with `namespace` and `qualifiedId`.
- **NamespacedIndexEntry**: Extends index entries with `namespace` and `qualifiedId`.

## Examples

### Basic Federation
This example demonstrates how to combine two separate knowledge bases and provide them to an agent.

```typescript
import { KnowledgeBase, FederatedKnowledgeBase } from 'yaaf/knowledge'
import { Agent } from 'yaaf'

const ml = await KnowledgeBase.load('./kb-ml')
const tools = await KnowledgeBase.load('./kb-tools')

const federated = FederatedKnowledgeBase.from({
  ml,
  tools,
})

const agent = new Agent({
  tools: federated.tools(),
  systemPrompt: federated.systemPromptSection(),
})
```

### Federation with Custom Labels
Labels can be provided to improve the agent's understanding of the knowledge source in the system prompt.

```typescript
const federated = FederatedKnowledgeBase.from({
  internal_docs: {
    kb: internalKb,
    label: "Company Internal Documentation (2023-2024)"
  },
  public_api: {
    kb: apiKb,
    label: "Public API Reference"
  }
})
```
