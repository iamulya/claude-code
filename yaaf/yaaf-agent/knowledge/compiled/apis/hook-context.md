## Overview
`HookContext` is a data structure provided to agent lifecycle hooks, such as `beforeToolCall` and `afterToolCall`. It contains the state of the current execution turn, including the tool being invoked, the arguments provided by the LLM, and the full conversation history. 

Unlike events, which are read-only notifications, hooks use this context to inform logic that can block, modify, or short-circuit the agent's execution loop.

## Signature / Constructor
```typescript
export type HookContext = {
  /** Name of the tool being called */
  toolName: string
  /** Parsed arguments the LLM passed to the tool */
  arguments: Record<string, unknown>
  /** Full conversation history at time of hook */
  messages: readonly ChatMessage[]
  /** Current iteration (LLM call count) in this agent run */
  iteration: number
}
```

## Methods & Properties
| Property | Type | Description |
| :--- | :--- | :--- |
| `toolName` | `string` | The unique identifier of the tool the agent is attempting to call. |
| `arguments` | `Record<string, unknown>` | A key-value map of the parameters generated by the LLM for the tool call. |
| `messages` | `readonly ChatMessage[]` | An immutable array containing the complete conversation history available to the agent at the moment the hook is triggered. |
| `iteration` | `number` | The current turn count of the LLM within the specific agent run, starting from 1. |

## Examples

### Inspecting Context in a Hook
This example demonstrates using `HookContext` to implement a manual confirmation step for specific tools.

```typescript
const agent = new Agent({
  systemPrompt: '...',
  hooks: {
    beforeToolCall: async (context: HookContext) => {
      const { toolName, arguments: args, iteration } = context;
      
      if (toolName === 'delete_database' && iteration < 5) {
        // Logic to check for user confirmation
        if (!userHasConfirmed) {
          return { action: 'block', reason: 'Manual confirmation required for destructive actions.' };
        }
      }
      
      return { action: 'continue' };
    },
  },
});
```

### Logging Tool Usage
Using the context to record tool execution metadata for auditing.

```typescript
const agent = new Agent({
  hooks: {
    afterToolCall: async (context: HookContext, result: any) => {
      await auditLog.record({
        tool: context.toolName,
        args: context.arguments,
        turn: context.iteration,
        output: result
      });
      return { action: 'continue' };
    },
  },
});
```
