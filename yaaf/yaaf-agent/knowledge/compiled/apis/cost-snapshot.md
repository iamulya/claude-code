## Overview

The `CostSnapshot` type defines the complete, serializable state of a `CostTracker` instance at a specific point in time. Its primary purpose is to enable the persistence of token usage and cost data, allowing an agent's session to be saved and later restored [Source 1].

An object of this type is generated by the `CostTracker.save()` method. This object can then be serialized (e.g., to JSON) and stored. Later, it can be used to recreate an identical `CostTracker` state using the static `CostTracker.restore()` method, preserving the accounting of tokens and costs across application restarts or different execution contexts [Source 1].

## Signature

`CostSnapshot` is a TypeScript `type` alias for an object with the following structure [Source 1]:

```typescript
export type CostSnapshot = {
  models: Record<string, ModelUsage>;
  totalCostUSD: number;
  totalInputTokens: number;
  totalOutputTokens: number;
  totalDurationMs: number;
  sessionId?: string;
  savedAt: string;
};
```

### Properties

*   **`models`**: `Record<string, ModelUsage>`
    A dictionary where keys are model identifier strings (e.g., `'gpt-4o'`) and values are `ModelUsage` objects containing detailed token counts, call counts, and cost for that specific model.
*   **`totalCostUSD`**: `number`
    The total estimated cost in USD, aggregated across all tracked models.
*   **`totalInputTokens`**: `number`
    The sum of all input tokens across all models.
*   **`totalOutputTokens`**: `number`
    The sum of all output tokens across all models.
*   **`totalDurationMs`**: `number`
    The total duration in milliseconds. The mechanism for tracking this duration is part of the `CostTracker` implementation.
*   **`sessionId`**: `string` (optional)
    An optional identifier for the session associated with this cost data.
*   **`savedAt`**: `string`
    An ISO 8601 timestamp string indicating [when](./when.md) the snapshot was created.

## Examples

The following example demonstrates the lifecycle of creating a `CostTracker`, saving its state to a `CostSnapshot`, and then restoring that state into a new instance.

```typescript
import { CostTracker, CostSnapshot } from 'yaaf';

// 1. Create and use a CostTracker instance
const tracker = new CostTracker();
tracker.record('gpt-4o', { inputTokens: 1000, outputTokens: 500 });
tracker.record('gpt-4o-mini', { inputTokens: 2000, outputTokens: 100 });

// 2. Save the current state to a snapshot object
// This object can be serialized to JSON and stored.
const snapshot: CostSnapshot = tracker.save();

console.log('Saved Snapshot:', JSON.stringify(snapshot, null, 2));
/*
Example output (costs are illustrative):
{
  "models": {
    "gpt-4o": {
      "inputTokens": 1000,
      "outputTokens": 500,
      "cacheReadTokens": 0,
      "cacheWriteTokens": 0,
      "costUSD": 0.0085,
      "calls": 1
    },
    "gpt-4o-mini": {
      "inputTokens": 2000,
      "outputTokens": 100,
      "cacheReadTokens": 0,
      "cacheWriteTokens": 0,
      "costUSD": 0.00045,
      "calls": 1
    }
  },
  "totalCostUSD": 0.00895,
  "totalInputTokens": 3000,
  "totalOutputTokens": 600,
  "totalDurationMs": 0,
  "savedAt": "2023-10-27T10:00:00.000Z"
}
*/

// 3. Restore the state into a new CostTracker instance
const restoredTracker = CostTracker.restore(snapshot);

console.log('Restored Total Cost:', restoredTracker.totalCostUSD);
// Output: Restored Total Cost: 0.00895

console.log('Restored GPT-4o calls:', restoredTracker.getModelUsage('gpt-4o').calls);
// Output: Restored GPT-4o calls: 1
```

## See Also

*   `CostTracker`: The class that manages and tracks token usage and costs, and which produces and consumes `CostSnapshot` objects.
*   `ModelUsage`: The type that defines the detailed usage statistics for a single model within the `models` property of a `CostSnapshot`.
