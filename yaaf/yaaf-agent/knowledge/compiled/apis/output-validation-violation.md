## Overview

The `OutputValidationViolation` type defines the structure of an object that describes a single, specific reason why a field in an [LLM](../concepts/llm.md)'s [Structured Output](../concepts/structured-output.md) failed validation. [when](./when.md) the `StructuredOutputValidator` class checks an LLM response against a set of `FieldRule` definitions, it generates an array of these objects for any discrepancies it finds.

This type is primarily used within the `OutputValidationEvent`, which is emitted by the validator to report on the outcome of a validation check. Each `OutputValidationViolation` provides detailed context about the failure, including which field was affected, what rule was broken, the actual value received, and the expected value, making it essential for debugging and handling invalid LLM outputs.

## Signature

`OutputValidationViolation` is a TypeScript type alias for an object with the following properties:

```typescript
export type OutputValidationViolation = {
  /** The name of the field that failed validation. Supports dot notation for nested objects (e.g., "user.address.zip"). */
  field: string;

  /** A string identifying the specific validation rule that was not met (e.g., "type", "required", "min", "pattern"). */
  rule: string;

  /** The actual value provided by the LLM for this field. */
  actual: unknown;

  /** A string describing the expected value or constraint (e.g., "number", "A value is required", "min: 0", "['red', 'green', 'blue']"). */
  expected: string;
};
```

## Examples

Below are examples of `OutputValidationViolation` objects that might be generated by the `StructuredOutputValidator`. These are typically found within the `violations` array of an `OutputValidationEvent`.

### Example 1: Wrong Data Type

If a field was expected to be a number but the LLM provided a string.

```typescript
const violation: OutputValidationViolation = {
  field: "product.price",
  rule: "type",
  actual: "ninety-nine",
  expected: "number",
};
```

### Example 2: Required Field Missing

If a field marked as `required` was not present in the LLM's output.

```typescript
const violation: OutputValidationViolation = {
  field: "user.email",
  rule: "required",
  actual: undefined,
  expected: "A value is required",
};
```

### Example 3: Number Out of Range

If a numeric field's value is less than the configured minimum.

```typescript
const violation: OutputValidationViolation = {
  field: "quantity",
  rule: "min",
  actual: -1,
  expected: "min: 0",
};
```

### Example 4: Value Not in Enum

If a string field's value is not one of the `allowedValues`.

```typescript
const violation: OutputValidationViolation = {
  field: "status",
  rule: "allowedValues",
  actual: "pending_review",
  expected: "['approved', 'rejected', 'pending']",
};
```

## See Also

- `StructuredOutputValidator`: The class that performs validation and generates these violations.
- `OutputValidationEvent`: The event object that contains an array of `OutputValidationViolation` instances.
