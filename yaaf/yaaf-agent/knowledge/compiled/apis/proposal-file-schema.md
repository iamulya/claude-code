## Overview

`ProposalFileSchema` is a Zod schema used within the YAAF [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md) to validate files containing proposed changes to the knowledge base [Ontology](../concepts/ontology.md) [Source 1]. It serves as a contract to ensure that suggestions for schema evolution, whether generated by an [LLM](../concepts/llm.md) or a user, are well-structured before being processed.

This schema validates an array of proposals, where each proposal specifies a kind of change, a confidence score, and a justification. The supported kinds of changes include adding new entity types, new [Vocabulary](../concepts/vocabulary.md) terms, new relationships between entity types, and new fields to existing entity types [Source 1].

## Signature

`ProposalFileSchema` is a `z.ZodObject` instance. The structure it validates is as follows:

```typescript
import { z } from "zod";

export const ProposalFileSchema = z.object({
  proposals: z.array(z.object({
    kind: z.enum(["add_entity_type", "add_vocabulary", "add_relationship", "add_field"]),
    confidence: z.number().min(0).max(1),
    reason: z.string(),
    entityType: z.string().optional(),
    term: z.string().optional(),
    from: z.string().optional(),
    to: z.string().optional(),
    details: z.record(z.string(), z.unknown()).optional(),
  })).default([]),
});
```

## Examples

### Validating a Proposal File

The following example demonstrates how to use `ProposalFileSchema` to parse and validate a data object containing various ontology change proposals.

```typescript
import { ProposalFileSchema } from './schemas';

const proposalData = {
  proposals: [
    {
      kind: "add_entity_type",
      confidence: 0.9,
      reason: "The source documents frequently mention 'API Endpoint' as a distinct concept.",
      entityType: "api_endpoint",
      details: {
        plural: "api_endpoints",
        description: "Represents a single callable endpoint in a web service."
      }
    },
    {
      kind: "add_vocabulary",
      confidence: 0.95,
      reason: "The term 'REST' is a common synonym for the 'restful_api' entity type.",
      entityType: "restful_api",
      term: "REST",
    },
    {
      kind: "add_relationship",
      confidence: 0.8,
      reason: "API endpoints are always part of a larger API service.",
      from: "api_endpoint",
      to: "api_service",
      details: {
        relationName: "belongs_to",
        cardinality: "many-to-one"
      }
    }
  ]
};

try {
  const validatedProposals = ProposalFileSchema.parse(proposalData);
  console.log("Validation successful:", validatedProposals);
} catch (error) {
  console.error("Validation failed:", error);
}
```
