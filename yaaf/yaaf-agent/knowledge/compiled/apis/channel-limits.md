## Overview

The `[[[[[[[[Channel]]]]]]]]Limits` type is a data structure that defines the technical constraints and formatting capabilities of a specific communication Channel, such as Telegram or Discord [Source 1]. It is used to ensure that messages generated by an agent can be successfully delivered through a given channel's transport layer.

This is particularly important for handling long agent responses. Components like the `Gateway` or utility functions like `chunkResponse` can use this information to automatically split a single large response into multiple smaller messages that respect the channel's `maxLength`, preventing delivery errors [Source 1].

## Signature

`ChannelLimits` is a TypeScript type alias for an object with the following properties [Source 1]:

```typescript
export type ChannelLimits = {
  /** Maximum message length in characters */
  maxLength: number;
  /** Supported markdown features */
  markdown?: boolean;
};
```

### Properties

- **`maxLength`**: `number`
  - The maximum number of characters allowed in a single message on this channel.

- **`markdown`**: `boolean` (optional)
  - Indicates whether the channel supports markdown formatting for text. If `undefined` or `false`, messages should be sent as plain text.

## Examples

### Defining Limits for a Custom Channel

[when](./when.md) implementing a custom `Channel`, you might define its limits to be used by response formatting logic.

```typescript
import { ChannelLimits } from 'yaaf';

// Example limits for a hypothetical "ChatterBox" platform
const chatterBoxLimits: ChannelLimits = {
  maxLength: 2000,
  markdown: true,
};

// Example limits for a simple SMS gateway
const smsGatewayLimits: ChannelLimits = {
  maxLength: 160,
  markdown: false,
};
```
