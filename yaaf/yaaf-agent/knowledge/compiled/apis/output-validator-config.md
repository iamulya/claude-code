## Overview
`OutputValidatorConfig` is a configuration interface used to define the behavior of the `StructuredOutputValidator`. It allows developers to enforce constraints on LLM-generated content, including JSON schema validation, character length limits, and URL safety checks. 

This configuration is primarily used to prevent downstream systems from processing malformed data, hallucinations, or potentially malicious URLs generated by an LLM.

## Signature / Constructor

```typescript
export type OutputValidatorConfig = {
  /**
   * Field validation rules.
   * Only applied when the LLM output contains parseable JSON.
   */
  rules?: FieldRule[]

  /**
   * Maximum output length in characters.
   * Default: 100_000.
   */
  maxOutputLength?: number

  /**
   * Action when validation fails:
   * - `warn` — log and pass through (default)
   * - `strip` — remove invalid fields
   * - `reject` — override the response with an error message
   */
  onViolation?: 'warn' | 'strip' | 'reject'

  /**
   * Called on each validation result.
   */
  onValidation?: (event: OutputValidationEvent) => void

  /**
   * Validate URLs in the output text (not just JSON).
   * When true, checks all URLs against allowedDomains or blocks known-dangerous ones.
   * Default: false.
   */
  validateUrls?: boolean

  /**
   * When validateUrls is true, only allow URLs from these domains.
   * If empty, all non-dangerous URLs are allowed.
   */
  allowedDomains?: string[]
}
```

### Related Types

#### FieldRule
Defines specific constraints for a JSON field.
```typescript
export type FieldRule = {
  field: string // Supports dot notation for nested fields (e.g., "user.email")
  type: 'string' | 'number' | 'boolean' | 'url' | 'email' | 'date' | 'enum'
  required?: boolean
  allowedValues?: unknown[] // For enum type
  min?: number // For number type
  max?: number // For number type
  maxLength?: number // For string length
  pattern?: RegExp // Regex pattern match
}
```

## Methods & Properties

| Property | Type | Description |
| :--- | :--- | :--- |
| `rules` | `FieldRule[]` | An array of rules to apply to JSON fields in the LLM output. |
| `maxOutputLength` | `number` | The maximum allowed character length for the LLM response. Defaults to `100,000`. |
| `onViolation` | `'warn' \| 'strip' \| 'reject'` | Determines the framework's response to a validation failure. Defaults to `'warn'`. |
| `onValidation` | `Function` | A callback function triggered after validation, receiving an `OutputValidationEvent`. |
| `validateUrls` | `boolean` | If `true`, the validator scans the entire output text for URLs and validates them. Defaults to `false`. |
| `allowedDomains` | `string[]` | A whitelist of domains permitted when `validateUrls` is enabled. |

## Examples

### Basic Schema Validation
This example configures the validator to ensure the LLM returns a valid "rating" number and a "category" from a specific list, rejecting the output if it fails.

```typescript
const config: OutputValidatorConfig = {
  rules: [
    { 
      field: "rating", 
      type: "number", 
      min: 1, 
      max: 5, 
      required: true 
    },
    { 
      field: "category", 
      type: "enum", 
      allowedValues: ["feedback", "inquiry", "complaint"] 
    }
  ],
  onViolation: "reject"
};
```

### URL Safety and Length Constraints
This example limits the output length and ensures any URLs mentioned in the text belong to trusted domains.

```typescript
const config: OutputValidatorConfig = {
  maxOutputLength: 5000,
  validateUrls: true,
  allowedDomains: ["docs.example.com", "api.example.com"],
  onViolation: "strip" // Removes invalid fields or URLs while keeping the rest
};
```
