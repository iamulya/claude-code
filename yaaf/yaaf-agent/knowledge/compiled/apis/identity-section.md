## Overview
The `identitySection` function is a utility used to define the core persona or identity of an LLM agent. It creates a structured segment intended for use within the `SystemPromptBuilder` assembly pipeline. 

Sections generated by this function are classified as "static," meaning they are computed once and cached for the lifetime of the agent's session. This design is optimized for performance and prompt-cache hits, as the agent's identity rarely changes during a continuous interaction.

## Signature
```typescript
export function identitySection(prompt: string): {
  /* internal section properties */
}
```

### Parameters
*   **prompt**: A string representing the agent's persona, instructions, or role (e.g., "You are a helpful coding assistant").

## Examples

### Basic Usage
The `identitySection` is typically used to encapsulate the primary instructions for an agent before they are added to a prompt builder.

```typescript
import { identitySection } from 'yaaf';

const identity = identitySection("You are a helpful coding assistant.");
```

### Integration with SystemPromptBuilder
While `identitySection` provides the structure, it is often utilized within a `SystemPromptBuilder` to manage the overall system prompt composition.

```typescript
import { SystemPromptBuilder, identitySection } from 'yaaf';

const builder = new SystemPromptBuilder()
  .addStatic('identity', () => 'You are a helpful coding assistant.');

// The identitySection function provides a standardized way to 
// define this content for the builder's registry.
```
