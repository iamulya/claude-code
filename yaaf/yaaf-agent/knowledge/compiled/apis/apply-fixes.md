## Overview

The `applyFixes` function is an asynchronous utility that processes a list of [Linting](../concepts/linting.md) issues and automatically applies fixes for specific, correctable problems within the compiled knowledge base articles [Source 1]. It is designed to improve the quality and consistency of the knowledge base by programmatically resolving common errors.

This function can automatically fix the following types of issues [Source 1]:
- `NON_CANONICAL_WIKILINK`: Rewrites a wikilink alias to its canonical title (e.g., `alias` becomes `Canonical Title`).
- `UNLINKED_MENTION`: Adds a wikilink to the first occurrence of a term that should be linked.
- `MISSING_REQUIRED_FIELD`: Injects a required field with a default value into an article's [Frontmatter](../concepts/frontmatter.md).

The fixer operates conservatively to ensure data integrity. It applies changes for one issue at a time, incrementally building the final version of the file. To optimize performance, it groups all changes for a single file and performs only one atomic write operation per file. If a fix for an issue cannot be applied because the target text has been altered by a previous fix in the same run, it skips that issue. The function is designed to only add or rewrite content; it never removes existing content [Source 1].

## Signature

```typescript
export async function applyFixes(
  issues: LintIssue[],
  compiledDir: string,
  dryRun: boolean = false,
): Promise<AutoFixResult>
```

**Parameters:**

- `issues: LintIssue[]`: An array of `LintIssue` objects, typically generated by the [Linter](../concepts/linter.md), that need to be processed for fixes [Source 1].
- `compiledDir: string`: The path to the directory containing the compiled article files that will be modified [Source 1].
- `dryRun: boolean = false`: An optional boolean that, [when](./when.md) set to `true`, prevents the function from writing any changes to the filesystem. This allows for previewing the intended modifications. It defaults to `false` [Source 1].

**Returns:**

- `Promise<AutoFixResult>`: A promise that resolves to an `AutoFixResult` object, which contains a summary of the fixes that were applied or would have been applied in a dry run [Source 1].

## Examples

### Applying Fixes to a Set of Issues

The following example demonstrates how to use `applyFixes` to correct a list of lint issues. The `dryRun` parameter is used to first preview and then apply the changes.

```typescript
import { applyFixes } from 'yaaf/knowledge';
import type { LintIssue } from 'yaaf/knowledge';

// Assume these issues were generated by the linter
const issues: LintIssue[] = [
  {
    code: 'NON_CANONICAL_WIKILINK',
    message: "Wikilink 'agent' should be 'Agent'",
    file: 'src/kb/articles/MyArticle.md',
    // ... other issue properties
    fix: {
      type: 'rewrite',
      range: [150, 159], // character range of 'agent'
      text: 'Agent',
    }
  },
  {
    code: 'UNLINKED_MENTION',
    message: "Unlinked mention of 'Plugin'",
    file: 'src/kb/articles/AnotherArticle.md',
    // ... other issue properties
    fix: {
      type: 'rewrite',
      range: [200, 206], // character range of 'Plugin'
      text: 'Plugin',
    }
  }
];

const compiledDir = './dist/kb';

async function runFixer() {
  // First, run in dry-run mode to see what would change
  console.log('--- Dry Run ---');
  const dryRunResult = await applyFixes(issues, compiledDir, true);
  console.log('Files to be changed:', dryRunResult.changedFiles);

  // If satisfied, run for real to apply the changes
  console.log('\n--- Applying Fixes ---');
  const finalResult = await applyFixes(issues, compiledDir, false);
  console.log('Fixes applied:', finalResult.fixedIssues.length);
  console.log('Files written:', finalResult.changedFiles);
}

runFixer();
```
