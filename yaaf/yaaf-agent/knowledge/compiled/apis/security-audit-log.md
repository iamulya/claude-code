## Overview
`SecurityAuditLog` is a centralized system for collecting and persisting security events generated by YAAF security middleware. It provides a structured, queryable audit trail designed for compliance, forensics, and threat intelligence.

The system supports severity classification, event correlation across sessions or users, and rolling retention to manage memory usage. It is typically used to monitor activities such as prompt injections, PII detection, and rate limiting violations.

## Signature / Constructor

The `SecurityAuditLog` can be instantiated directly or via the `securityAuditLog` factory function.

### Factory Function
```typescript
export function securityAuditLog(config?: AuditLogConfig): SecurityAuditLog;
```

### Configuration Types

#### AuditLogConfig
| Property | Type | Description |
| :--- | :--- | :--- |
| `maxEntries` | `number` | Maximum entries to retain in memory before evicting oldest. Default: 10,000. |
| `onEntry` | `(entry: AuditEntry) => void` | Real-time callback for forwarding entries to external systems (SIEM, etc.). |
| `minSeverity` | `AuditSeverity` | Minimum severity level to log. Default: 'info'. |
| `sessionId` | `string` | Optional session ID for event correlation. |

#### AuditEntry
Each log entry contains the following structure:
```typescript
export type AuditEntry = {
  id: string;
  timestamp: Date;
  severity: AuditSeverity;
  category: AuditCategory;
  summary: string;
  data?: Record<string, unknown>;
  userId?: string;
  sessionId?: string;
  source: string;
}
```

#### AuditSeverity
*   `info`
*   `warning`
*   `critical`

#### AuditCategory
Supported categories include: `prompt_injection`, `output_sanitized`, `pii_detected`, `pii_redacted`, `trust_violation`, `grounding_failed`, `rate_limited`, `input_anomaly`, `access_denied`, `tool_blocked`, `canary_triggered`, and `custom`.

## Methods & Properties

Based on the framework's architecture, the `SecurityAuditLog` provides the following capabilities:

*   **Event Recording**: Appends structured security events to the log.
*   **Retention Management**: Automatically evicts old entries based on the `maxEntries` configuration.
*   **Statistics**: Generates `AuditStats` including entry counts by severity, category, and source, as well as identifying top users and time ranges.
*   **Exporting**: Supports exporting the audit trail to JSON or NDJSON formats.
*   **Real-time Forwarding**: Triggers the `onEntry` callback for integration with external logging providers or SIEMs.

## Examples

### Basic Initialization
```typescript
import { securityAuditLog } from 'yaaf/security';

const auditLog = securityAuditLog({
  maxEntries: 5000,
  minSeverity: 'warning',
  onEntry: (entry) => {
    console.log(`Security Event [${entry.severity}]: ${entry.summary}`);
  }
});
```

### Monitoring Statistics
```typescript
// Assuming auditLog is an instance of SecurityAuditLog
const stats = auditLog.getStats();

console.log(`Total security events: ${stats.totalEntries}`);
console.log(`Critical events: ${stats.bySeverity.critical}`);
```
