## Overview

The `GroundingAssessment` type defines the structure of the detailed report generated by a `GroundingValidator` after it evaluates an [LLM](../concepts/llm.md)'s response for factual grounding [Source 1]. This object serves as the primary output for understanding the validator's findings.

It is most commonly accessed via the `onAssessment` callback in the `GroundingValidatorConfig`. This allows for logging, monitoring, or custom logic based on the outcome of each grounding check [Source 1]. The assessment includes an overall score, counts of checked and grounded sentences, the action taken by the validator, and a detailed per-sentence analysis [Source 1].

## Signature

`GroundingAssessment` is a TypeScript type alias. Its structure includes an array of `GroundingSentence` objects, which are also defined below [Source 1].

```typescript
export type GroundingAssessment = {
  /** Overall [[[[[[[[Grounding Score]]]]]]]] (0-1) */
  score: number;
  /** Number of factual sentences checked */
  totalSentences: number;
  /** Number of grounded sentences */
  groundedSentences: number;
  /** Per-sentence breakdown */
  sentences: GroundingSentence[];
  /** Action taken */
  action: "passed" | "warned" | "annotated" | "overridden";
  /** Timestamp */
  timestamp: Date;
};

export type GroundingSentence = {
  /** The sentence text */
  text: string;
  /** Whether this sentence is grounded in tool evidence */
  grounded: boolean;
  /** Number of overlapping tokens with tool evidence */
  overlapCount: number;
  /** Which tool result provided the best evidence (if any) */
  bestSource?: string;
  /**
   * Which method determined grounding.
   * 'keyword' = TF-IDF overlap; 'embedding' = cosine similarity; 'llm' = LLM scorer.
   */
  scoredBy?: "keyword" | "embedding" | "llm";
};
```

## Properties

### `GroundingAssessment` Properties

*   **`score`**: `number`
    An overall Grounding Score between 0 and 1, representing the fraction of factual sentences that were successfully grounded in the provided evidence [Source 1].
*   **`totalSentences`**: `number`
    The total number of sentences in the LLM response that were identified as factual claims and subjected to a grounding check. Short sentences like greetings are typically excluded [Source 1].
*   **`groundedSentences`**: `number`
    The number of sentences from `totalSentences` that were successfully verified against the evidence from [tool results](../concepts/tool-results.md) [Source 1].
*   **`sentences`**: `GroundingSentence[]`
    An array providing a detailed breakdown of the assessment for each individual sentence that was checked [Source 1].
*   **`action`**: `"passed" | "warned" | "annotated" | "overridden"`
    A string indicating the action taken by the `GroundingValidator` based on its configuration and the assessment score. For example, `"warned"` if the mode is `warn` and ungrounded sentences were found, or `"overridden"` if the mode is `strict` and the score was below the `minCoverage` threshold [Source 1].
*   **`timestamp`**: `Date`
    The `Date` object representing [when](./when.md) the assessment was performed [Source 1].

### `GroundingSentence` Properties

*   **`text`**: `string`
    The original text of the sentence being evaluated [Source 1].
*   **`grounded`**: `boolean`
    A boolean indicating `true` if the sentence was successfully grounded in evidence, and `false` otherwise [Source 1].
*   **`overlapCount`**: `number`
    The number of significant keywords that overlap between the sentence and the best-matching piece of evidence [Source 1].
*   **`bestSource`**: `string` (optional)
    If the sentence is grounded, this property may contain the text of the tool result that provided the strongest evidence [Source 1].
*   **`scoredBy`**: `"keyword" | "embedding" | "llm"` (optional)
    Indicates which scoring mechanism made the final determination for this sentence's grounding status. This is useful for understanding the validator's reasoning process [Source 1].

## Examples

The most common way to interact with a `GroundingAssessment` is by providing an `onAssessment` callback when configuring a `GroundingValidator`. This allows for logging or other side effects based on the validation result.

```typescript
import { GroundingValidator, GroundingAssessment } from 'yaaf';

// A callback function to handle the assessment report
function logAssessment(assessment: GroundingAssessment) {
  console.log(`Grounding check complete. Action: ${assessment.action}`);
  console.log(`Score: ${assessment.score.toFixed(2)} (${assessment.groundedSentences}/${assessment.totalSentences} sentences grounded)`);

  for (const sentence of assessment.sentences) {
    const status = sentence.grounded ? '✅ GROUNDED' : '⚠️ UNGROUNDED';
    console.log(`- [${status}] "${sentence.text}" (Scored by: ${sentence.scoredBy || 'N/A'})`);
  }
}

// Configure the validator to use the callback
const validator = new GroundingValidator({
  mode: 'warn',
  onAssessment: logAssessment,
});

// This validator would now be added to an agent's hooks.
// When the agent runs and the validator hook is triggered,
// the `logAssessment` function will be called with the results.
```
