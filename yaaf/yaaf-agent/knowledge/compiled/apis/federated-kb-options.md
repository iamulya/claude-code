## Overview

`FederatedKBOptions` is a TypeScript type that defines optional configuration parameters for a `FederatedKnowledgeBase` instance [Source 1]. A `FederatedKnowledgeBase` combines multiple [Knowledge Base](../subsystems/knowledge-base.md) instances, and these options allow for customizing the behavior of the tools it generates and managing the size of its contribution to the agent's [System Prompt](../concepts/system-prompt.md) [Source 1].

This configuration is typically provided when creating a `FederatedKnowledgeBase` to control aspects like the number of search results returned by its tools or to adjust the [Token Budget](../concepts/token-budget.md) allocated for describing the federated namespaces in the system prompt [Source 1].

## Signature

The `FederatedKBOptions` type is defined as an object with the following optional properties [Source 1]:

```typescript
export type FederatedKBOptions = {
  /**
   * Options passed to the generated tools.
   */
  toolOptions?: KBToolOptions;

  /**
   * Maximum token budget for the system prompt KB section.
   * Same progressive degradation as KnowledgeBaseOptions.systemPromptMaxTokens.
   * Default: 3000 tokens (federations have more namespaces to describe).
   */
  systemPromptMaxTokens?: number;
};
```

### Properties

*   `toolOptions?`: An optional [KBToolOptions](./kb-tool-options.md) object. These options are passed directly to the tools generated by the `FederatedKnowledgeBase`, such as `search_kb` and `fetch_kb_document`, allowing for customization of their behavior (e.g., the number of search results to return) [Source 1].
*   `systemPromptMaxTokens?`: An optional `number` that specifies the maximum [Token Budget](../concepts/token-budget.md) for the [System Prompt](../concepts/system-prompt.md) section that describes the federated knowledge bases. If the description exceeds this budget, it undergoes progressive degradation to fit. The default value is 3000 tokens, which is higher than for a single `KnowledgeBase` to accommodate the descriptions of multiple namespaces [Source 1].

## Examples

The following example demonstrates how to provide `FederatedKBOptions` when creating a `FederatedKnowledgeBase` to customize its behavior.

```typescript
import {
  KnowledgeBase,
  FederatedKnowledgeBase,
  FederatedKBOptions,
} from 'yaaf/knowledge';

// Assume kbML and kbTools are pre-existing, loaded KnowledgeBase instances
const kbML = await KnowledgeBase.load('./kb-ml');
const kbTools = await KnowledgeBase.load('./kb-tools');

// Define custom options for the federated knowledge base
const options: FederatedKBOptions = {
  // Set a larger token budget for the system prompt section
  systemPromptMaxTokens: 4000,
  // Configure the generated tools to return up to 5 search results
  toolOptions: {
    searchResultCount: 5,
  },
};

// Create the federated knowledge base with the specified options
const federated = FederatedKnowledgeBase.from(
  {
    ml: kbML,
    tools: kbTools,
  },
  options,
);

// The tools and system prompt generated by `federated` will now respect these options.
const agentTools = federated.tools();
const systemPromptSection = federated.systemPromptSection();
```
