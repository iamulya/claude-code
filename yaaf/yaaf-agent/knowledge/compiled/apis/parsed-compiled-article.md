## Overview
`ParsedCompiledArticle` is a data structure used within the Knowledge Base (KB) compilation and linting pipeline. It represents a markdown article that has been read from the filesystem and processed into a normalized format. 

This type serves as the primary input for linting checks, providing a structured view of an article's metadata (frontmatter) and content (body) alongside its filesystem location and unique identifier. It is typically produced by the `readCompiledArticles` or `parseCompiledArticle` functions.

## Signature / Constructor
```typescript
export type ParsedCompiledArticle = {
  /** Relative docId (no .md extension), e.g. "concepts/attention-mechanism" */
  docId: string
  /** Absolute path to the compiled file */
  filePath: string
  /** Parsed frontmatter key-value pairs */
  frontmatter: Record<string, unknown>
  /** Markdown body after the frontmatter block */
  body: string
}
```

## Methods & Properties
*   **docId**: A string representing the relative identifier of the document. It does not include the `.md` file extension (e.g., `subsystems/agent-runtime`).
*   **filePath**: A string containing the absolute path to the compiled markdown file on the local disk.
*   **frontmatter**: An object containing key-value pairs extracted from the YAML frontmatter block at the start of the markdown file.
*   **body**: A string containing the remaining markdown content of the article after the frontmatter block has been stripped.

## Examples
### Manual Construction
While usually generated by the framework's reader, a `ParsedCompiledArticle` follows this structure:

```typescript
const article: ParsedCompiledArticle = {
  docId: "concepts/llm-integration",
  filePath: "/path/to/project/knowledge/compiled/concepts/llm-integration.md",
  frontmatter: {
    title: "LLM Integration",
    entity_type: "concept",
    category: "core"
  },
  body: "\n## Overview\nThis article describes how LLMs are integrated..."
};
```

### Usage in Linting
The type is used to iterate over articles during validation phases:

```typescript
function checkArticleTitle(article: ParsedCompiledArticle) {
  if (!article.frontmatter.title) {
    console.error(`Article ${article.docId} is missing a title in frontmatter.`);
  }
}
```
