## Overview

The `DataScope` type represents the resolved data access scope for a given request [Source 1]. It is the output of a `DataScopeStrategy` and is passed to [Tools](../subsystems/tools.md) via the `ToolExecutionContext`. Its primary purpose is to instruct tools on how to filter their data access, such as by adding `WHERE` clauses to database queries or passing filter parameters to external API calls [Source 1].

A `DataScope` contains a set of filters, a flag to indicate unrestricted access, and metadata for auditing purposes. This mechanism is central to implementing security patterns like multi-tenancy, user-based resource ownership, and attribute-based access control ([abac](./abac.md)) at the data layer [Source 1].

## Signature

`DataScope` is a TypeScript type alias with the following structure [Source 1]:

```typescript
export type DataScope = {
  /**
   * The name of the DataScopeStrategy that produced this scope.
   */
  strategy: string;

  /**
   * A record of filters to apply. Tools are responsible for interpreting
   * these filters and applying them as WHERE clauses, API filter parameters, etc.
   */
  filters: Record<string, unknown>;

  /**
   * When true, this scope allows unrestricted access, and tools can
   * bypass data filtering entirely.
   */
  unrestricted?: boolean;

  /**
   * A human-readable description of the scope, primarily for use in audit logs.
   * e.g., "Scoped to tenant acme-corp"
   */
  description?: string;
};
```

## Examples

### Multi-Tenant Scoping

A `DataScope` generated by a `TenantScopeStrategy` to isolate data for a specific tenant. A tool would use `filters.tenantId` in its database queries.

```typescript
import { DataScope } from 'yaaf';

const tenantScope: DataScope = {
  strategy: 'TenantScopeStrategy',
  filters: {
    tenantId: 'acme-corp',
  },
  unrestricted: false,
  description: 'Scoped to tenant acme-corp',
};
```

### User Ownership Scoping

A `DataScope` that restricts a tool to only access resources owned by the current user.

```typescript
import { DataScope } from 'yaaf';

const userOwnershipScope: DataScope = {
  strategy: 'OwnershipScopeStrategy',
  filters: {
    ownerId: 'alice-123',
  },
  unrestricted: false,
  description: "Filtered to resources owned by user 'alice-123'",
};
```

### Unrestricted Admin Scope

A `DataScope` for an administrative user who has unrestricted access to all data. Tools should check the `unrestricted` flag before applying any filters.

```typescript
import { DataScope } from 'yaaf';

const adminScope: DataScope = {
  strategy: 'AdminRoleStrategy',
  filters: {},
  unrestricted: true,
  description: 'Unrestricted access granted to admin user',
};
```
