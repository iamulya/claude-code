## Overview

The `DedupResult` interface defines the shape of the object returned by the `deduplicatePlans` function. It represents the outcome of the [Semantic Deduplication](../concepts/semantic-deduplication.md) stage (P4-1) within the [Knowledge Compiler](../subsystems/knowledge-compiler.md) [Source 1].

This structure cleanly separates the article plans that survived the deduplication process from those that were identified as duplicates and merged into other plans. It provides a clear record of which plans were removed and which survivor they were absorbed into, which is useful for debugging and understanding the compilation pipeline [Source 1].

## Signature

```typescript
import type { ArticlePlan } from "./extractor/index.js";

export interface DedupResult {
  /** Surviving plans after deduplication */
  merged: ArticlePlan[];
  /** Plans that were absorbed into survivors */
  removed: Array<{ docId: string; mergedInto: string; similarity: number }>;
}
```
[Source 1]

## Properties

### merged

- **Type:** `ArticlePlan[]`

An array of `ArticlePlan` objects that survived the deduplication process. These are the unique, consolidated plans that will proceed to the next stage of the knowledge compilation pipeline. If a plan in this array absorbed other plans, its contents (like source paths) will have been updated to reflect the merge [Source 1].

### removed

- **Type:** `Array<{ docId: string; mergedInto: string; similarity: number }>`

An array of objects that serves as a log for all plans that were removed during deduplication. Each object contains:
- `docId`: The unique identifier of the plan that was removed.
- `mergedInto`: The `docId` of the surviving plan that this plan was absorbed into.
- `similarity`: The calculated similarity score that triggered the merge [Source 1].

## Examples

The following example demonstrates the structure of a `DedupResult` object returned by the `deduplicatePlans` function.

```typescript
import { deduplicatePlans, DedupResult } from 'yaaf';
import type { ArticlePlan } from 'yaaf';

// Assume these plans were generated by the extractor
const planA: ArticlePlan = {
  docId: 'bert-arch',
  title: 'BERT Architecture',
  description: 'A deep dive into the Bidirectional Encoder Representations from Transformers model.',
  sourcePaths: ['source1.txt'],
  // ... other properties
};

const planB: ArticlePlan = {
  docId: 'bidirectional-encoder',
  title: 'Bidirectional Encoder Representations',
  description: 'Explaining the pre-trained language model known as BERT.',
  sourcePaths: ['source2.txt'],
  // ... other properties
};

const planC: ArticlePlan = {
  docId: 'gpt-model',
  title: 'The GPT Model',
  description: 'An overview of the Generative Pre-trained Transformer.',
  sourcePaths: ['source3.txt'],
  // ... other properties
};

const allPlans: ArticlePlan[] = [planA, planB, planC];

// The deduplication function processes the plans
const result: DedupResult = deduplicatePlans(allPlans, 0.7);

/*
Expected result:

result.merged would contain two ArticlePlan objects:
1. A modified version of planA (docId: 'bert-arch') which now also includes
   sourcePaths from planB.
2. The original planC (docId: 'gpt-model'), which was not a duplicate.

result.removed would contain one entry:
{
  docId: 'bidirectional-encoder',
  mergedInto: 'bert-arch',
  similarity: 0.85 // example similarity score
}
*/

console.log(JSON.stringify(result, null, 2));
```
