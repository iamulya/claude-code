## Overview

`Task[[[[[[[[Notification]]]]]]]]` is a TypeScript type that defines the structured data payload a `WorkerAgent` sends to a `CoordinatorAgent` upon completing a delegated task [Source 1]. It is a fundamental component of YAAF's coordinator-worker multi-agent pattern.

This type ensures that communication from workers to the coordinator is consistent and contains all necessary information about a task's outcome, including its status, a summary, an optional detailed result, and performance metrics [Source 1]. The `CoordinatorAgent` uses these notifications to track progress, synthesize results, and decide on next steps.

## Signature

`TaskNotification` is an object type with the following structure [Source 1]:

```typescript
export type TaskStatus = "completed" | "failed" | "killed";

export type TaskNotification = {
  taskId: string;
  status: TaskStatus;
  summary: string;
  result?: string;
  usage?: {
    totalTokens: number;
    toolUses: number;
    durationMs: number;
  };
};
```

### Fields

*   **`taskId: string`**
    A unique identifier for the task that was executed.

*   **`status: TaskStatus`**
    The final status of the task. `TaskStatus` is a union type that can be one of `"completed"`, `"failed"`, or `"killed"`.

*   **`summary: string`**
    A concise, human-readable summary of the task's outcome. This is often used by the `CoordinatorAgent` to understand the result without parsing the full output.

*   **`result?: string`** (optional)
    The detailed, complete output or result generated by the worker agent.

*   **`usage?: object`** (optional)
    An object containing performance and cost metrics for the task.
    *   `totalTokens: number`: The total number of [LLM](../concepts/llm.md) tokens consumed during the task.
    *   `toolUses: number`: The number of times [Tools](../subsystems/tools.md) were invoked.
    *   `durationMs: number`: The total execution time of the task in milliseconds.

## Examples

### Basic Notification

A `TaskNotification` object representing a successfully completed research task.

```typescript
import type { TaskNotification } from 'yaaf';

const notification: TaskNotification = {
  taskId: 'research-task-123',
  status: 'completed',
  summary: 'Successfully retrieved the top 3 competitors for YAAF from public sources.',
  result: '1. LangChain\n2. LlamaIndex\n3. Autogen',
  usage: {
    totalTokens: 1520,
    toolUses: 3,
    durationMs: 4500,
  }
};
```

### Handling Notifications in a Coordinator

A `CoordinatorAgent` can listen for events that carry a `TaskNotification` payload to react to worker progress.

```typescript
import { createCoordinator } from 'yaaf';
import type { TaskNotification } from 'yaaf';

// Assume model and workers are defined elsewhere
const coordinator = createCoordinator({
  model,
  workers: [ /* ... */ ],
});

coordinator.on('worker:completed', (notification: TaskNotification) => {
  console.log(`Worker task ${notification.taskId} finished with status: ${notification.status}`);
  console.log(`Summary: ${notification.summary}`);
});

// When the coordinator runs and delegates a task, the 'worker:completed'
// event will fire with a TaskNotification payload upon completion.
await coordinator.run('Research the agent framework market.');
```
