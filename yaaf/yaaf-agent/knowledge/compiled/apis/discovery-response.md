## Overview

The `[[Discovery]]Response` type defines the structure for the output of a [Knowledge Base Discovery](../subsystems/knowledge-base-discovery.md) process. This process analyzes the existing knowledge base to identify potential gaps, such as missing articles or weak connections between existing articles.

This type is derived from the `DiscoveryResponseSchema`, a Zod schema used within the YAAF [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md) to validate data at its boundaries [Source 1]. It serves as a structured report that can be used to guide the creation of new content or the improvement of existing links within the knowledge graph.

## Signature

`DiscoveryResponse` is a TypeScript type inferred from a Zod schema. Its structure is defined by `DiscoveryResponseSchema` [Source 1].

The underlying Zod schema is defined as follows:

```typescript
import { z } from "zod";

export const DiscoveryResponseSchema = z.object({
  missingArticles: z.array(z.object({
    title: z.string().min(1),
    entityType: z.string().min(1),
    reason: z.string().min(1),
  })).default([]),
  weakConnections: z.array(z.object({
    fromDocId: z.string().min(1),
    toDocId: z.string().min(1),
    reason: z.string().min(1),
  })).default([]),
});
```

The resulting TypeScript type is:

type DiscoveryResponse = {
  missingArticles: {
    title: string;
    entityType: string;
    reason: string;
  }[];
  weakConnections: {
    fromDocId: string;

## Properties

The `DiscoveryResponse` type has the following properties:

| Property          | Type                                                              | Description                                                                                                                            |
| ----------------- | ----------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `missingArticles` | `Array<{ title: string; entityType: string; reason: string; }>`   | An array of objects, each identifying a potentially missing article. Includes the suggested `title`, `entityType`, and a `reason` for its inclusion. |
| `weakConnections` | `Array<{ fromDocId: string; toDocId: string; reason: string; }>` | An array of objects, each identifying a weak or missing link between two existing articles, identified by their document IDs (`fromDocId`, `toDocId`). |

## Examples

The following is an example of a `DiscoveryResponse` object that might be generated by a [Discovery](../concepts/discovery.md) process. It identifies one missing article and one weak connection between two existing articles.

```typescript
import type { DiscoveryResponse } from 'yaaf';

const discoveryResult: DiscoveryResponse = {
  missingArticles: [
    {
      title: "Attention Mechanism",
      entityType: "concept",
      reason: "Referenced by 'Transformer Architecture' and 'BERT' but does not exist."
    }
  ],
  weakConnections: [
    {
      fromDocId: "concepts/transformer-architecture",
      toDocId: "apis/agent",
      reason: "The 'Agent' article mentions using a Transformer model, but does not link to the 'Transformer Architecture' concept article."
    }
  ]
};

// This result can now be used to plan the creation of a new article
// and to suggest an update to an existing one.
console.log(discoveryResult.missingArticles[0].title); // "Attention Mechanism"
```
