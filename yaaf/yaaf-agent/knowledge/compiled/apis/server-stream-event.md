## Overview
`ServerStreamEvent` defines the structure of messages emitted by the YAAF server's streaming endpoint (`POST /chat/stream`). It provides a standardized format for real-time communication between the agent and a client, covering incremental text generation, tool execution lifecycle, and final execution metadata such as token usage.

This type is primarily used by developers building client-side integrations or custom UI components that need to parse and react to the agent's step-by-step progress.

## Signature / Constructor

```typescript
export type ServerStreamEvent = {
  type: 'text_delta' | 'tool_call_start' | 'tool_call_result' | 'done'
  /** Text content for text_delta */
  text?: string
  /** Alias for text (some agents use 'content' instead) */
  content?: string
  /** Tool name for tool events */
  toolName?: string
  /** Tool call error flag */
  error?: boolean
  /** Tool call duration */
  durationMs?: number
  /** Token usage — carried by 'done' events */
  usage?: {
    promptTokens: number
    completionTokens: number
    cacheReadTokens?: number
  }
}
```

## Properties

### type
The discriminator for the event type.
- `text_delta`: Indicates a fragment of text has been generated.
- `tool_call_start`: Indicates the agent has initiated a tool call.
- `tool_call_result`: Indicates a tool call has completed.
- `done`: Indicates the agent has finished the entire request.

### text / content
The string fragment generated by the LLM. Both fields are provided for compatibility; `text` is the standard field, while `content` serves as an alias. These are only present when `type` is `text_delta`.

### toolName
The name of the tool being invoked or completed. This is present when `type` is `tool_call_start` or `tool_call_result`.

### error
A boolean flag indicating if a tool execution encountered an error. This is present when `type` is `tool_call_result`.

### durationMs
The time in milliseconds taken to execute a tool. This is present when `type` is `tool_call_result`.

### usage
An object containing token consumption statistics. This is typically only provided when `type` is `done`.
- `promptTokens`: Number of tokens in the input.
- `completionTokens`: Number of tokens in the output.
- `cacheReadTokens`: (Optional) Number of tokens retrieved from cache.

## Examples

### Handling a Stream Event
In a client-side application, you might process incoming SSE messages like this:

```typescript
function handleStreamMessage(event: ServerStreamEvent) {
  switch (event.type) {
    case 'text_delta':
      console.log('New text:', event.text);
      break;
    case 'tool_call_start':
      console.log(`Starting tool: ${event.toolName}`);
      break;
    case 'tool_call_result':
      console.log(`Tool ${event.toolName} finished in ${event.durationMs}ms`);
      break;
    case 'done':
      console.log('Finished. Total tokens:', event.usage?.promptTokens);
      break;
  }
}
```
