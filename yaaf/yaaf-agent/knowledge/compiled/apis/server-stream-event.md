## Overview

The `ServerStreamEvent` type defines the structure of JSON objects sent over a Server-Sent Events (SSE) connection from a YAAF agent server. This is the standard format for events from the `/chat/stream` endpoint created by the [createServer](./create-server.md) function [Source 1].

Clients consuming a streaming agent response will receive a series of these events, each describing a step in the agent's execution, such as generating text, calling a tool, or finishing the run. The `type` property acts as a discriminant to determine the nature of the event [Source 1].

## Signature

`ServerStreamEvent` is a TypeScript type alias for an object with the following structure [Source 1]:

```typescript
export type ServerStreamEvent = {
  type: "text_delta" | "tool_call_start" | "tool_call_result" | "done";
  /** Text content for text_delta */
  text?: string;
  /** Alias for text (some agents use 'content' instead) */
  content?: string;
  /** Tool name for tool events */
  toolName?: string;
  /** Tool call error flag */
  error?: boolean;
  /** Tool call duration */
  durationMs?: number;
  /** Token usage — carried by 'done' events */
  usage?: {
    promptTokens: number;
    completionTokens: number;
    cacheReadTokens?: number;
  };
};
```

### Properties

| Property         | Type                                                              | Description                                                                                                                                                           |
| ---------------- | ----------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `type`           | `"text_delta" \| "tool_call_start" \| "tool_call_result" \| "done"` | **(Required)** The type of event. This determines which other properties are present.                                                                                 |
| `text`           | `string`                                                          | Optional. The chunk of text generated by the agent. Present for `text_delta` events.                                                                                  |
| `content`        | `string`                                                          | Optional. An alias for `text`.                                                                                                                                        |
| `toolName`       | `string`                                                          | Optional. The name of the tool being called or that has finished executing. Present for `tool_call_start` and `tool_call_result` events.                             |
| `error`          | `boolean`                                                         | Optional. A flag indicating if a tool call resulted in an error. Present for `tool_call_result` events.                                                              |
| `durationMs`     | `number`                                                          | Optional. The execution duration of a tool in milliseconds. Present for `tool_call_result` events.                                                                    |
| `usage`          | `object`                                                          | Optional. An object detailing token usage for the entire agent run. This is only present on the final `done` event.                                                   |
| `usage.promptTokens` | `number`                                                          | The number of tokens in the input prompt.                                                                                                                             |
| `usage.completionTokens` | `number`                                                          | The number of tokens generated in the completion.                                                                                                                     |
| `usage.cacheReadTokens` | `number`                                                          | Optional. The number of tokens served from a cache, if applicable.                                                                                                    |

## Examples

A client connected to the `/chat/stream` endpoint might receive a sequence of events like the following during a single agent run. Note that in a real SSE stream, each JSON object would be prefixed with `data: `.

```json
// First, a chunk of text is generated
{
  "type": "text_delta",
  "text": "Okay, I will search for that. "
}

// The agent decides to use a tool
{
  "type": "tool_call_start",
  "toolName": "webSearch"
}

// The tool finishes successfully
{
  "type": "tool_call_result",
  "toolName": "webSearch",
  "durationMs": 450,
  "error": false
}

// The agent generates more text based on the tool result
{
  "type": "text_delta",
  "text": "The search results indicate..."
}

// The agent run is complete
{
  "type": "done",
  "usage": {
    "promptTokens": 50,
    "completionTokens": 120
  }
}
```
