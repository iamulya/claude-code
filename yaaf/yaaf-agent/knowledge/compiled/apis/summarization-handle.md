## Overview
The `SummarizationHandle` is a control object returned by the `startAgentSummarization` function. It is used to manage the lifecycle of a background summarization process, which periodically generates short progress updates (e.g., "Reading config.json") for an agent's activity. 

This handle allows developers to stop the background loop when an agent completes its task and provides a synchronous way to retrieve the most recently generated summary for display in user interfaces or logs.

## Signature / Constructor
```typescript
export type SummarizationHandle = {
  /** Stop the periodic summarization loop. */
  stop: () => void
  /** Get the most recent summary, or null if none generated yet. */
  lastSummary: () => string | null
}
```

## Methods & Properties
### stop
`stop: () => void`

Terminates the periodic summarization loop. Once called, the background process will no longer schedule new summarization tasks. This should be called when the agent finishes its execution to prevent resource leaks.

### lastSummary
`lastSummary: () => string | null`

Returns the most recent summary text generated by the model. If the summarization loop has not yet produced a result (for instance, if the `minMessages` threshold has not been reached or the first model call is still in progress), this method returns `null`.

## Examples
The following example demonstrates starting a summarization loop for a worker agent and using the handle to stop it once the worker's task is complete.

```typescript
import { startAgentSummarization } from './agentSummary.js';

const summaryHandle = startAgentSummarization({
  agentId: 'worker-1',
  model: smallFastModel,
  getMessages: () => worker.messages,
  onSummary: (text) => updateUI(`Worker 1: ${text}`),
});

// Perform agent work...
await worker.run();

// Retrieve the final summary if needed
const finalStatus = summaryHandle.lastSummary();
console.log(`Final status: ${finalStatus}`);

// Stop the background loop
summaryHandle.stop();
```
