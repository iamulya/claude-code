## Overview

`ExtractionArticleSchema` is a Zod schema used within the YAAF [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md) to validate the structure and content of a single article proposal generated by an [LLM](../concepts/llm.md) [Source 1].

During the knowledge extraction phase, an LLM analyzes source material and proposes a plan for creating or updating articles in the knowledge base. Each proposed article in this plan must conform to the `ExtractionArticleSchema`. This schema acts as a contract at the boundary between the framework and untrusted LLM output, ensuring that all incoming data is well-formed before further processing [Source 1].

It is a key component of the `ExtractionPlanSchema`, which validates an array of these article proposals [Source 1].

## Signature

`ExtractionArticleSchema` is a `z.ZodObject` instance with the following structure [Source 1]:

```typescript
import { z } from "zod";

export const ExtractionArticleSchema = z.object({
  // The proposed canonical title for the article.
  canonicalTitle: z.string().min(1).max(200),

  // The entity type of the proposed article.
  entityType: z.string().min(1),

  // The action to be taken for this article.
  action: z.enum(["create", "update", "merge", "skip"]),

  // An optional, LLM-suggested document ID. Must follow a specific format.
  docIdSuggestion: z
    .string()
    .regex(/^[a-z][a-z0-9\-\/]+$/)
    .optional(),

  // The LLM's confidence in this proposal, from 0.0 to 1.0.
  confidence: z.number().min(0).max(1).default(0.5),

  // An array of source identifiers this proposal is derived from. Must not be empty.
  sources: z.array(z.string()).min(1),

  // An array of document IDs for articles related to this one.
  relatedArticles: z.array(z.string()).default([]),

  // An array of document IDs for articles that reference this one.
  referencedBy: z.array(z.string()).default([]),
});
```

## Examples

### Validating a valid article proposal

The following example demonstrates how to use `ExtractionArticleSchema` to successfully parse a valid article proposal object.

```typescript
import { ExtractionArticleSchema } from "./schemas";

const validProposal = {
  canonicalTitle: "Agent",
  entityType: "concept",
  action: "create",
  docIdSuggestion: "concepts/agent",
  confidence: 0.95,
  sources: ["source-doc-001.txt"],
  relatedArticles: ["api/agent-class"],
  referencedBy: [],
};

try {
  const parsedProposal = ExtractionArticleSchema.parse(validProposal);
  console.log("Validation successful:", parsedProposal);
} catch (error) {
  console.error("Validation failed:", error);
}
```

### Validating an invalid article proposal

This example shows how the schema will throw an error [when](./when.md) parsing an object with missing or incorrectly formatted fields.

const invalidProposal = {

  action: "create",
  confidence: 1.1, // Invalid: > 1.0
  sources: [], // Invalid: must have at least one source
};

try {
  const parsedProposal = ExtractionArticleSchema.parse(invalidProposal);
  console.log("Validation successful:", parsedProposal);
} catch (error) {
  console.error("Validation failed:", error.errors);
}
