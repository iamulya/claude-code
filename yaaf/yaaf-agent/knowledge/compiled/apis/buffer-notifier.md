## Overview

The `BufferNotifier` is a class that implements the `Notification[[[[[[[[Channel]]]]]]]]` interface. Instead of sending Notifications to an external service like a webhook or console, it collects them in an internal, in-[Memory](../concepts/memory.md) buffer [Source 1].

This makes it particularly useful for two main scenarios:
1.  **Testing**: In unit or integration tests, `BufferNotifier` can be used to capture Notifications generated by an agent or orchestrator. Test assertions can then be made against the buffered [Notification](./notification.md)s to verify that the correct events were emitted.
2.  **Batch Processing**: In situations where notifications should be aggregated and processed together at a later time, `BufferNotifier` can serve as a temporary holding area.

## Signature / Constructor

`BufferNotifier` implements the `NotificationChannel` interface. The source material does not specify a constructor, implying a default, parameterless constructor.

```typescript
import type { NotificationChannel, Notification } from 'yaaf';

export class BufferNotifier implements NotificationChannel {
  /**
   * Adds the notification to an internal in-memory buffer.
   * This method does not throw.
   */
  notify(notification: Notification): Promise<void>;

  /**
   * Optional cleanup method.
   */
  destroy?(): Promise<void>;
}
```

## Methods & Properties

### notify()

Adds a `Notification` object to the internal buffer. This method is asynchronous and returns a `Promise<void>`. It is designed not to throw errors [Source 1].

-   **Signature**: `notify(notification: Notification): Promise<void>`
-   **Parameters**:
    -   `notification`: The `Notification` object to be stored.
-   **Returns**: `Promise<void>`

## Examples

// In a test file

  it('should emit a "completed" notification', async () => {
    const bufferNotifier = new BufferNotifier();

      type: 'completed',
      title: 'Agent Task Finished',
      message: 'The analysis is complete.',
