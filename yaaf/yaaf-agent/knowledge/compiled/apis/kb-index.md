## Overview

The `KBIndex` type defines the structure of the top-level index for a YAAF Knowledge Base (KB). It serves as a manifest, providing a high-level summary of the KB's contents, including aggregate statistics and a complete list of all documents [Source 1].

This object is typically generated by the `KBStore` and can be used to create an `llms.txt`-style index file, which provides context to an [LLM](../concepts/llm.md) about the available knowledge documents [Source 1]. It contains the total number of documents, an estimate of the total token count, a description of the domain, and an array of `KBIndexEntry` objects, one for each document in the KB [Source 1].

## Signature

`KBIndex` is a TypeScript type that describes the overall KB manifest. It relies on the `KBIndexEntry` type for individual document listings [Source 1].

```typescript
export type KBIndex = {
  /** Domain description from [[[[[[[[Ontology]]]]]]]] */
  domain?: string;
  /** Total number of documents */
  totalDocuments: number;
  /** Total estimated tokens across all documents */
  totalTokenEstimate: number;
  /** Index entries for each document */
  entries: KBIndexEntry[];
};

export type KBIndexEntry = {
  docId: string;
  title: string;
  entityType: string;
  isStub: boolean;
  /** One-line summary (first sentence of body) */
  summary: string;
};
```

## Properties

### Properties of `KBIndex`

*   `domain?: string`
    An optional string providing a high-level description of the knowledge base's domain, typically sourced from the project's Ontology [Source 1].

*   `totalDocuments: number`
    The total number of documents contained within the knowledge base [Source 1].

*   `totalTokenEstimate: number`
    The sum of estimated tokens across all documents in the knowledge base [Source 1].

*   `entries: KBIndexEntry[]`
    An array of `KBIndexEntry` objects, where each object represents a single document in the knowledge base [Source 1].

### Properties of `KBIndexEntry`

*   `docId: string`
    The unique identifier for the document, typically derived from its file path (e.g., `concepts/agent-lifecycle`) [Source 1].

*   `title: string`
    The canonical title of the article, usually from its [Frontmatter](../concepts/frontmatter.md) [Source 1].

*   `entityType: string`
    The type of the entity the document describes (e.g., `concept`, `api`, `guide`), as defined in the ontology [Source 1].

*   `isStub: boolean`
    A boolean flag indicating whether the document is a stub or a complete article [Source 1].

*   `summary: string`
    A one-line summary of the document, typically extracted from the first sentence of its body text [Source 1].

## Examples

The following is an example of a `KBIndex` object for a small knowledge base.

```typescript
import type { KBIndex } from 'yaaf';

const exampleIndex: KBIndex = {
  domain: "A TypeScript-first, provider-agnostic, production-grade agent framework.",
  totalDocuments: 152,
  totalTokenEstimate: 85340,
  entries: [
    {
      docId: "concepts/agent-lifecycle",
      title: "Agent Lifecycle",
      entityType: "concept",
      isStub: false,
      summary: "The agent lifecycle describes the sequence of states an agent transitions through from creation to termination."
    },
    {
      docId: "api/Agent",
      title: "Agent",
      entityType: "api",
      isStub: false,
      summary: "The main class for creating and running an LLM-powered agent."
    },
    {
      docId: "guides/creating-a-plugin",
      title: "Creating a Plugin",
      entityType: "guide",
      isStub: true,
      summary: "This guide walks through the process of building a custom plugin for YAAF."
    }
  ]
};
```
