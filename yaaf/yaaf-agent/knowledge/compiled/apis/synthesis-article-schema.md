## Overview

`SynthesisArticleSchema` is a Zod schema used within the YAAF [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md) to validate the structure of a complete article generated by an [LLM](../concepts/llm.md) [Source 1]. It serves as a data contract at the boundary between the compiler and the untrusted output from a language model's synthesis step.

This schema specifically validates the metadata wrapper of the generated article, such as its ID, title, and sources. It ensures that the core fields are present and correctly formatted before the article is processed further or written to the knowledge base. The content of the markdown body itself is checked for minimum length but not for its internal structure [Source 1].

## Signature

`SynthesisArticleSchema` is a `const` export of a Zod object schema.

```typescript
import { z } from "zod";

export const SynthesisArticleSchema = z.object({
  docId: z.string().min(1),
  title: z.string().min(1),
  entityType: z.string().min(1),
  markdown: z.string().min(10),
  confidence: z.number().min(0).max(1).default(0.5),
  sources: z.array(z.string()).default([]),
});
```

### Fields

*   `docId`: A non-empty string representing the unique document identifier.
*   `title`: A non-empty string for the article's title.
*   `entityType`: A non-empty string specifying the article's [Entity Type](../concepts/entity-type.md) (e.g., 'api', 'concept').
*   `markdown`: A string containing the full markdown content of the article, which must be at least 10 characters long.
*   `confidence`: A number between 0.0 and 1.0 indicating the LLM's confidence in the generated content. Defaults to `0.5`.
*   `sources`: An array of strings listing the source identifiers used to generate the article. Defaults to an empty array `[]`.

### Inferred Type

The TypeScript type inferred from this schema is `SynthesisArticle`.

## Examples

### Validating a valid article object

The following example shows how to use `SynthesisArticleSchema` to parse a valid object representing an LLM's output.

```typescript
import { SynthesisArticleSchema } from 'yaaf';

const llmOutput = {
  docId: 'api/agent/create-agent',
  title: 'createAgent',
  entityType: 'api',
  markdown: '## Overview\n\nThe `createAgent` function initializes a new agent...',
  confidence: 0.95,
  sources: ['source/src/agent.ts'],
};

try {
  const validatedArticle = SynthesisArticleSchema.parse(llmOutput);
  console.log('Validation successful:', validatedArticle);
} catch (error) {
  console.error('Validation failed:', error);
}
```

### Validating an invalid article object

If the input object does not conform to the schema (e.g., a required field is missing), the `.parse()` method will throw a `ZodError`.

```typescript
import { SynthesisArticleSchema } from 'yaaf';

const invalidLlmOutput = {
  docId: 'api/agent/create-agent',
  // Missing 'title' and 'entityType'
  markdown: 'This is too short', // Fails min(10)
  sources: ['source/src/agent.ts'],
};

try {
  const validatedArticle = SynthesisArticleSchema.parse(invalidLlmOutput);
} catch (error) {
  console.error('Validation failed:', error.errors);
  /*
  Outputs an array of validation errors, for instance:
  [
    {
      "code": "invalid_type",
      "expected": "string",
      "received": "undefined",
      "path": ["title"],
      "message": "Required"
    },
    {
      "code": "too_small",
      "minimum": 10,
      "type": "string",
      "inclusive": true,
      "path": ["markdown"],
      "message": "String must contain at least 10 character(s)"
    },
    ...
  ]
  */
}
```
