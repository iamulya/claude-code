## Overview

The `postProcessCompiledArticles` function is an integral part of the YAAF knowledge base compilation pipeline [Source 1]. It executes two key transformations on all synthesized articles after they have been generated by an [LLM](../concepts/llm.md) but before they are checked by the [Linter](../concepts/linter.md) [Source 1].

Its primary responsibilities are:

1.  **[Wikilink Resolution](../concepts/wikilink-resolution.md)**: It scans article content for `wikilink` syntax and converts them into standard relative markdown links (e.g., `[display text](../path/to/doc.md)`). This process uses the `ConceptRegistry` to find the correct path for each link target. [Wikilinks](../concepts/wikilinks.md) that cannot be found in the registry are left unchanged for the linter to flag as errors [Source 1].

2.  **[Article Segmentation](../concepts/article-segmentation.md)**: It identifies articles that exceed a specified [Token Budget](../concepts/token-budget.md) and splits them into smaller, sequentially linked sub-articles. The split occurs at H2 (`##`) section boundaries, and navigation links (`[ŌåÉ Previous]`, `[Next ŌåÆ]`) are automatically inserted to connect the parts [Source 1].

These operations are configurable and can be enabled or disabled via the `options` parameter [Source 1].

## Signature

The function is asynchronous and orchestrates the post-processing of an entire directory of compiled articles.

```typescript
export async function postProcessCompiledArticles(
  compiledDir: string,
  registry: ConceptRegistry,
  options?: PostProcessOptions
): Promise<PostProcessResult>;
```

### Parameters

-   `compiledDir` (`string`): The file path to the directory containing the compiled markdown articles to be processed [Source 1].
-   `registry` (`ConceptRegistry`): The [Concept Registry](../subsystems/concept-registry.md) instance used to resolve wikilink targets to their corresponding file paths [Source 1].
-   `options` (`PostProcessOptions`, optional): An object to configure the post-processing behavior [Source 1].

### Configuration (`PostProcessOptions`)

The `options` parameter allows for fine-grained control over the post-processing steps.

```typescript
export type PostProcessOptions = {
  /** 
   * The maximum number of tokens an article can have before it is segmented.
   * @default 15000 
   */
  tokenBudget?: number;

  /** 
   * If true, wikilink resolution will be performed.
   * @default true 
   */
  resolveLinks?: boolean;

  /** 
   * If true, oversized articles will be segmented.
   * @default true 
   */
  segmentArticles?: boolean;
};
```

### Return Value (`PostProcessResult`)

The function returns a `Promise` that resolves to a `PostProcessResult` object, summarizing the outcome of the operations.

```typescript
export type PostProcessResult = {
  /** Statistics on resolved and unresolved wikilinks. */
  wikilinks: { resolved: number; unresolved: number };

  /** Result of the segmentation process, or null if not run. */
  segmentation: SegmentResult | null;

  /**
   * A mapping of each article's docId to the docIds of articles it links to.
   * This is used by the differential manifest to track dependencies.
   */
  wikilinkDeps: Record<string, string[]>;

  /**
   * A list of docIds for articles that still contain unresolved wikilinks.
   * This helps the differential compiler target refreshes when new concepts are added.
   */
  unresolvedDocIds: string[];
};
```

## Examples

The following example demonstrates how to invoke `postProcessCompiledArticles` as part of a larger build script.

```typescript
import { postProcessCompiledArticles } from 'yaaf';
import type { ConceptRegistry } from 'yaaf';
import { loadConceptRegistry } from './ontology-loader'; // A hypothetical loader

async function runCompilerPipeline() {
  const compiledArticlesDir = './dist/compiled';
  
  // Load the registry which maps concepts to file paths
  const registry: ConceptRegistry = await loadConceptRegistry();

  console.log('Running post-processing on compiled articles...');
  
  try {
    const result = await postProcessCompiledArticles(
      compiledArticlesDir, 
      registry, 
      {
        tokenBudget: 10000, // Use a custom token budget
        resolveLinks: true,
        segmentArticles: true,
      }
    );

    console.log(`Post-processing complete.`);
    console.log(`- Resolved wikilinks: ${result.wikilinks.resolved}`);
    
    if (result.wikilinks.unresolved > 0) {
      console.warn(`- Unresolved wikilinks: ${result.wikilinks.unresolved}`);
      console.warn(`  - Docs with unresolved links: ${result.unresolvedDocIds.join(', ')}`);
    }

    if (result.segmentation && result.segmentation.splitCount > 0) {
      console.log(`- Segmented ${result.segmentation.splitCount} oversized articles.`);
    }

  } catch (error) {
    console.error('Failed during post-processing:', error);
  }
}

runCompilerPipeline();
```
