## Overview

The `[[[[[[[[Discovery]]]]]]]]Suggestion` type defines the structure for an object that recommends the creation of a new article in the knowledge base [Source 1]. These suggestions are a key output of the [LLM](../concepts/llm.md)-powered Discovery mode, which analyzes the entire knowledge base to find structural gaps and areas for improvement [Source 1].

A `DiscoverySuggestion` is generated [when](./when.md) the discovery process identifies a concept that is mentioned frequently across existing articles but does not have its own dedicated article. It provides a proposed title, [Entity Type](../concepts/entity-type.md), a justification for its creation, and a count of how many times the concept was referenced [Source 1].

These objects are contained within the `missingArticles` array of the `DiscoveryResult` type, which is the return value of the `discoverGaps` function [Source 1].

## Signature

`DiscoverySuggestion` is a TypeScript type alias with the following structure:

```typescript
export type DiscoverySuggestion = {
  /** Suggested article title */
  title: string;
  /** Suggested entity type */
  entityType: string;
  /** Why this article should exist */
  reason: string;
  /** How many existing articles mention this concept */
  mentionCount: number;
};
```

### Properties

| Property       | Type     | Description                                                  |
| -------------- | -------- | ------------------------------------------------------------ |
| `title`        | `string` | The suggested title for the new article [Source 1].          |
| `entityType`   | `string` | The suggested entity type for the new article [Source 1].    |
| `reason`       | `string` | An explanation, typically generated by an LLM, of why this article should be created [Source 1]. |
| `mentionCount` | `number` | The number of existing articles that reference this concept [Source 1]. |

## Examples

The primary use of `DiscoverySuggestion` is to process the results from the `discoverGaps` function. The following example shows how to iterate through the `missingArticles` array and display the details of each suggestion.

```typescript
import { discoverGaps, makeKBLLMClient } from 'yaaf/knowledge';
// Assume 'compiledDir', 'registry', and 'ontology' are defined

const llm = makeKBLLMClient();
const result = await discoverGaps(llm, compiledDir, registry, ontology);

console.log("Found missing article suggestions:");
for (const suggestion of result.missingArticles) {
  console.log(
    `- Title: ${suggestion.title}\n` +
    `  Reason: ${suggestion.reason}\n` +
    `  Mentioned In: ${suggestion.mentionCount} other articles\n`
  );
}

/*
Example Output:
Found missing article suggestions:
- Title: Agent State Management
  Reason: The concept of 'state' is mentioned frequently in articles about Agent, AgentContext, and long-running conversations, but there is no central article explaining state management strategies.
  Mentioned In: 8 other articles
*/
```
