## Overview

`VigilEvents` is a TypeScript mapped type that defines the complete set of events emitted by a `Vigil` agent instance. It maps event names (e.g., `'tick'`, `'error'`) to the specific shape of their data payloads. This type is used to provide strong typing for event listeners, ensuring that callback functions receive the correct arguments [when](./when.md) handling events from the autonomous agent.

The events cover the entire lifecycle and operation of a `Vigil` agent, including:
- **Lifecycle:** `start`, `stop`
- **Autonomous Operations:** `tick`, `cron:fire`
- **Agent Output:** `brief`
- **Operational Health:** `cron:delayed`, `error`

Developers use these event types implicitly when subscribing to events via the `Vigil` class's `on()` method.

## Signature

The `VigilEvents` type is defined as an object where keys are event names and values are their corresponding payload types.

```typescript
export type VigilEvents = {
  /** Agent processed a tick (proactive wake-up interval) */
  tick: { count: number; response: string };

  /** Cron task fired and was dispatched to the agent */
  "cron:fire": { task: ScheduledTask; response: string };

  /**
   * Cron task was delayed due to a busy agent (exponential back-off).
   * Observable so operators can detect tasks that are consistently deferred.
   */
  "cron:delayed": { taskId: string; retryCount: number; delayMs: number; reason: string };

  /** Agent produced structured output via the brief [[[[[[[[Channel]]]]]]]] */
  brief: { message: string; timestamp: Date };

  /** Tick, cron task, persistence, or watchdog operation failed */
  error: { source: "tick" | "cron" | "persist" | "watchdog"; error: Error; task?: ScheduledTask };

  /** Vigil started [[[[[[[[Autonomous Loop]]]]]]]] */
  start: { tickInterval: number; taskCount: number };

  /** Vigil stopped */
  stop: { ticksRun: number; tasksRun: number };
};
```

## Event Payloads

This section details the payload for each event defined in `VigilEvents`.

### `tick`
Fired each time the agent's Autonomous Loop "ticks" based on its `tickInterval`. This signifies a proactive wake-up.

- **`count`**: `number` - The sequential number of this tick since the agent started.
- **`response`**: `string` - The full response generated by the agent in response to the tick prompt.

### `cron:fire`
Fired when a scheduled cron task is due and has been dispatched to the agent for execution.

- **`task`**: `ScheduledTask` - The full scheduled task object that was executed.
- **`response`**: `string` - The full response generated by the agent in response to the task's prompt.

### `cron:delayed`
Fired when a scheduled cron task was due but could not be executed immediately because the agent was already busy. The task will be retried with an exponential back-off. This event is useful for monitoring agent load and detecting tasks that are consistently deferred.

- **`taskId`**: `string` - The ID of the delayed task.
- **`retryCount`**: `number` - The number of times this task has been retried.
- **`delayMs`**: `number` - The duration in milliseconds the task will wait before the next attempt.
- **`reason`**: `string` - A message explaining why the task was delayed (e.g., "Agent is busy").

### `brief`
Fired when the agent produces a structured message through its "brief" output Channel. This is the primary way for an autonomous agent to communicate results or status updates.

- **`message`**: `string` - The content of the agent's output.
- **`timestamp`**: `Date` - The time at which the brief was generated.

### `error`
Fired when an unhandled error occurs within one of Vigil's subsystems.

- **`source`**: `"tick" | "cron" | "persist" | "watchdog"` - The subsystem where the error originated.
- **`error`**: `Error` - The underlying `Error` object.
- **`task`**: `ScheduledTask` (optional) - If the error originated from a cron task, this property will contain the task object.

### `start`
Fired once when the `Vigil` agent's autonomous loop successfully starts after `agent.start()` is called.

- **`tickInterval`**: `number` - The configured tick interval in milliseconds.
- **`taskCount`**: `number` - The number of scheduled tasks loaded at startup.

### `stop`
Fired once when the `Vigil` agent's autonomous loop has fully stopped after `agent.stop()` is called.

- **`ticksRun`**: `number` - The total number of ticks processed during the session.
- **`tasksRun`**: `number` - The total number of cron tasks fired during the session.

## Examples

The following example demonstrates how to subscribe to several events on a `Vigil` instance. TypeScript's type inference, powered by `VigilEvents`, ensures the `data` parameter in each callback is correctly typed.

```typescript
import { Vigil } from 'yaaf';

const agent = new Vigil({
  systemPrompt: 'You are a proactive assistant.',
  tickInterval: 60_000, // Tick every minute
});

// Schedule a recurring task
agent.schedule('0 * * * *', 'Check for new PR reviews.');

// Listen for the 'start' event
agent.on('start', ({ tickInterval, taskCount }) => {
  console.log(`Vigil agent started with tick interval ${tickInterval}ms and ${taskCount} tasks.`);
});

// Listen for each autonomous 'tick'
agent.on('tick', ({ count, response }) => {
  console.log(`Tick #${count} completed.`);
  // You might log the response or take other actions
});

// Listen for cron tasks firing
agent.on('cron:fire', ({ task }) => {
  console.log(`Cron task fired: ${task.id} ("${task.prompt}")`);
});

// Listen for agent output
agent.on('brief', ({ message, timestamp }) => {
  console.log(`[${timestamp.toISOString()}] Agent says: ${message}`);
});

// Listen for errors
agent.on('error', ({ source, error, task }) => {
  console.error(`Error in ${source}:`, error);
  if (task) {
    console.error(`Error occurred during task: ${task.id}`);
  }
});

await agent.start();
```
