## Overview
`parseArticleOutput` is a utility function used within the YAAF knowledge compiler's synthesis pipeline. Its primary responsibility is to decompose a raw markdown string generated by a Large Language Model (LLM) into its constituent parts: the YAML frontmatter block and the markdown body content.

The function is designed to handle the standard markdown convention where frontmatter is enclosed between triple-dash (`---`) delimiters at the start of the file. It is built to be resilient; if the LLM omits the frontmatter block despite prompting, the function returns an empty frontmatter object rather than throwing an error.

## Signature
```typescript
export function parseArticleOutput(raw: string): ParsedArticle
```

### Parameters
- `raw`: The raw string output received from the LLM, expected to contain a YAML frontmatter block followed by markdown content.

### Return Value
Returns a `ParsedArticle` object. Based on the source material, this object contains:
- A record of frontmatter fields.
- The remaining markdown body string.

## Examples

### Basic Usage
This example demonstrates parsing a standard LLM response containing both frontmatter and a body.

```typescript
import { parseArticleOutput } from 'yaaf/knowledge/compiler/synthesizer/frontmatter';

const rawOutput = `---
title: Agent Architecture
entity_type: concept
summary: An overview of agent structures.
---
# Agent Architecture
Agents in YAAF are composed of several subsystems...`;

const parsed = parseArticleOutput(rawOutput);

console.log(parsed.frontmatter.title); // "Agent Architecture"
console.log(parsed.body); // "# Agent Architecture\nAgents in YAAF..."
```

### Handling Missing Frontmatter
If the input lacks the triple-dash delimiters, the function treats the entire input as the body.

```typescript
const rawOutput = "Just a plain markdown string without frontmatter.";
const parsed = parseArticleOutput(rawOutput);

// parsed.frontmatter is an empty object {}
// parsed.body contains the full original string
```
