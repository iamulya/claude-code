## Overview

The `FieldRule` type defines a single validation constraint for a specific field within a structured (JSON) output from a Large Language Model ([LLM](../concepts/llm.md)). It is a core component of the `StructuredOutputValidator` subsystem, which uses an array of `FieldRule` objects to create a schema for validating and sanitizing LLM-generated data [Source 1].

Each `FieldRule` specifies the field's name, expected data type, and other constraints such as whether it is required, its maximum length, a numeric range, or a set of allowed values. This allows developers to enforce a strict structure on agent outputs, preventing malformed or unexpected data from being processed by downstream systems [Source 1].

## Signature

`FieldRule` is a TypeScript type alias.

```typescript
export type FieldRule = {
  /** Field name (supports dot notation for nested: "address.city") */
  field: string;
  /** Expected type */
  type: FieldType;
  /** Whether the field is required */
  required?: boolean;
  /** Allowed values (for enum type) */
  allowedValues?: unknown[];
  /** Minimum value (for number type) */
  min?: number;
  /** Maximum value (for number type) */
  max?: number;
  /** Maximum string length */
  maxLength?: number;
  /** Regex pattern the value must match */
  pattern?: RegExp;
};
```

### Properties

| Property        | Type                                                              | Description                                                                                             |
| --------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `field`         | `string`                                                          | The name of the field to validate. Supports dot notation for accessing nested object properties, e.g., `"user.address.zipcode"` [Source 1]. |
| `type`          | `FieldType`                                                       | The expected data type of the field. `FieldType` can be `"string"`, `"number"`, `"boolean"`, `"url"`, `"email"`, `"date"`, or `"enum"` [Source 1]. |
| `required`      | `boolean` (optional)                                              | If `true`, the field must be present in the output. Defaults to `false` if not specified [Source 1].    |
| `allowedValues` | `unknown[]` (optional)                                            | An array of permitted values. This is used [when](./when.md) `type` is `"enum"` [Source 1].                          |
| `min`           | `number` (optional)                                               | The minimum allowed value for a field of `type` `"number"` [Source 1].                                  |
| `max`           | `number` (optional)                                               | The maximum allowed value for a field of `type` `"number"` [Source 1].                                  |
| `maxLength`     | `number` (optional)                                               | The maximum allowed length for a field of `type` `"string"` [Source 1].                                 |
| `pattern`       | `RegExp` (optional)                                               | A regular expression that the field's value must match [Source 1].                                      |

## Examples

The following example shows how to define an array of `FieldRule` objects to validate a user profile generated by an LLM. These rules are typically passed as part of an `OutputValidatorConfig`.

```typescript
import { FieldRule, OutputValidatorConfig } from 'yaaf';

// Define a set of validation rules for a user profile object
const userProfileRules: FieldRule[] = [
  {
    field: 'name',
    type: 'string',
    required: true,
    maxLength: 50,
  },
  {
    field: 'age',
    type: 'number',
    required: false,
    min: 18,
    max: 120,
  },
  {
    // Use dot notation for nested fields
    field: 'account.type',
    type: 'enum',
    required: true,
    allowedValues: ['free', 'premium', 'enterprise'],
  },
  {
    field: 'contact.email',
    type: 'email',
    required: true,
  },
  {
    field: 'website',
    type: 'url',
    required: false,
  },
  {
    field: 'postalCode',
    type: 'string',
    // Enforce a US ZIP code format
    pattern: /^[0-9]{5}(-[0-9]{4})?$/,
  }
];

// These rules are used within the configuration for StructuredOutputValidator
const validatorConfig: OutputValidatorConfig = {
  rules: userProfileRules,
  onViolation: 'reject', // Reject any LLM output that violates the rules
};
```
