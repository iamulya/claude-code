## Overview
`ArticlePlan` represents a discrete instruction generated by the Concept Extractor during the knowledge compilation process. It defines how one or more source files should be synthesized into a single Knowledge Base (KB) article. 

An `ArticlePlan` is produced deterministically by the Extractor. A single source file (such as a comprehensive research paper) may result in multiple `ArticlePlan` objects if it covers multiple distinct concepts. Conversely, multiple source files may be grouped into a single `ArticlePlan` if they contribute information to the same canonical entity. These plans are subsequently processed by the Knowledge Synthesizer to create or update Markdown files in the KB.

## Signature / Constructor
```typescript
export type ArticlePlan = {
  docId: string
  canonicalTitle: string
  entityType: string
  action: ArticleAction
  existingDocId?: string
  sourcePaths: string[]
  knownLinkDocIds: string[]
  candidateNewConcepts: CandidateConcept[]
  suggestedFrontmatter: Record<string, unknown>
  skipReason?: string
  confidence: number
}
```

## Methods & Properties
- **docId**: The target identifier for the article, relative to the compiled directory and excluding the `.md` extension. It is formatted as `{entityType}/{slug-of-canonical-title}` (e.g., `concepts/attention-mechanism`). This value is computed deterministically.
- **canonicalTitle**: The primary title of the article.
- **entityType**: The specific entity type key derived from the ontology.
- **action**: The operation to be performed by the Synthesizer. Supported values include:
    - `create`: Generate a new article because no matching entry exists in the registry.
    - `update`: Merge new information into an existing article.
    - `skip`: Disregard the source material (used for non-KB-worthy content like licenses or changelogs).
- **existingDocId**: If the `action` is `update`, this field contains the `docId` of the existing article in the registry.
- **sourcePaths**: An array of absolute file paths for the ingested content that contributes to this article.
- **knownLinkDocIds**: A list of `docId`s for existing KB articles that should be referenced via wikilinks within the generated article.
- **candidateNewConcepts**: A list of `CandidateConcept` objects representing new entities discovered in the source that do not yet have KB articles.
- **suggestedFrontmatter**: A collection of metadata field values inferred from the source. These are tagged as "compiler-inferred" and do not overwrite explicit author-provided frontmatter.
- **skipReason**: A string explaining why an article was marked with the `skip` action, used for compilation reporting.
- **confidence**: A numerical score between 0 and 1 representing the system's confidence in the entity classification. Plans with a confidence score below 0.5 are typically flagged for human review.

## Examples
### Creating a New Article Plan
This example demonstrates a plan to create a new article about the "Transformer" architecture based on a source PDF.

```typescript
const transformerPlan: ArticlePlan = {
  docId: "concepts/transformer-architecture",
  canonicalTitle: "Transformer Architecture",
  entityType: "concept",
  action: "create",
  sourcePaths: ["/abs/path/to/attention-is-all-you-need.pdf"],
  knownLinkDocIds: ["concepts/neural-networks", "concepts/attention-mechanism"],
  candidateNewConcepts: [
    {
      name: "Multi-Head Attention",
      entityType: "concept",
      description: "A specific attention mechanism used in Transformers.",
      mentionCount: 5
    }
  ],
  suggestedFrontmatter: {
    category: "architecture",
    tags: ["llm", "deep-learning"]
  },
  confidence: 0.98
};
```
