## Overview

The `loadProposals` function reads and validates [Ontology](../concepts/ontology.md) proposals from the `.kb-ontology-proposals.json` file within a specified knowledge base directory. This function is a key part of the [Ontology Feedback Loop](../concepts/ontology-feedback-loop.md) (O7), allowing [Tools](../subsystems/tools.md) to consume suggestions previously generated by the [Knowledge Base Compiler](../subsystems/knowledge-base-compiler.md) [Source 1].

A critical feature of this function is its built-in validation. Before returning the proposals, it checks the structure of each one. Any proposals that are malformed, have invalid confidence values, unknown kinds, or potentially malicious keys (e.g., for prototype pollution) are silently discarded. This ensures that only safe and correctly structured proposals are processed, which is especially important [when](./when.md) proposals might be automatically applied to the ontology [Source 1].

## Signature

The function takes the path to the knowledge base directory and returns a promise that resolves to an array of valid `OntologyProposal` objects [Source 1].

```typescript
export async function loadProposals(kbDir: string): Promise<OntologyProposal[]>;
```

### Parameters

-   `kbDir` (string): The file system path to the root of the knowledge base directory. The function will look for a `.kb-ontology-proposals.json` file inside this directory.

### Return Value

-   `Promise<OntologyProposal[]>`: A promise that resolves to an array of `OntologyProposal` objects. If the proposals file does not exist or contains no valid proposals, it resolves to an empty array.

### Related Types

The `OntologyProposal` type defines the structure of a single suggestion for evolving the [Knowledge Base Ontology](../subsystems/knowledge-base-ontology.md) [Source 1].

```typescript
export type OntologyProposal = {
  kind: "add_entity_type" | "add_vocabulary" | "add_relationship" | "add_field";
  confidence: number; // 0–1
  reason: string;
  /** For add_entity_type */
  entityType?: string;
  /** For add_vocabulary */
  term?: string;
  /** For add_relationship */
  from?: string;
  to?: string;
  /** Extra structured data */
  details?: Record<string, unknown>;
};
```

## Examples

The following example demonstrates how to load ontology proposals from a knowledge base located at `./my-kb`.

```typescript
import { loadProposals } from 'yaaf';
import { join } from 'path';

async function processOntologySuggestions() {
  const kbDirectory = './my-kb';

  try {
    const proposals = await loadProposals(kbDirectory);

    if (proposals.length > 0) {
      console.log(`Loaded ${proposals.length} valid ontology proposals.`);
      // Example: Log high-confidence proposals
      proposals
        .filter(p => p.confidence > 0.9)
        .forEach(p => console.log(`High confidence proposal: ${p.reason}`));
    } else {
      console.log('No valid ontology proposals found.');
    }
  } catch (error) {
    // This might happen if the directory is unreadable, etc.
    // Note: A missing or empty proposals file resolves to [] and does not throw.
    console.error('Failed to load proposals:', error);
  }
}

processOntologySuggestions();
```
